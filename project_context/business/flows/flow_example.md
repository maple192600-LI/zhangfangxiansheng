# 手工快速录入（示例）

> 这是一个示例条目，展示 flow 类型文件的写法。业务人员后续可在前端创建正式流程条目。

## 基本信息

| 字段 | 内容 |
|------|------|
| 标题 | 手工快速录入 |
| 所属模块 | 流水录入 |
| 一句话说明 | 通过可编辑表格逐笔录入流水，系统自动匹配单位和账户后写入 FundEvent |

## 流程关系

| 字段 | 内容 |
|------|------|
| 上一步 | 主数据维护（需要已有法人和账户） |
| 下一步 | 预览确认（自动跳转） |
| 输入来自哪里 | 用户在可编辑表格中逐行填写 |
| 输出到哪里 | FundEvent（流水事实表） |

## 影响范围

| 字段 | 内容 |
|------|------|
| 影响页面 | ManualFlow.vue、UploadPreview.vue |
| 影响报表 | 保存后立即可在所有报表中查到 |

## 状态

| 字段 | 内容 |
|------|------|
| 当前状态 | 可用 |

## 业务人员说明

选择"快速录入"模式 → 点击"添加行" → 逐行填写单位简称、账户编号、日期、摘要、金额 → 点击"保存到暂存区" → 自动跳转预览页面。匹配不到法人或账户的行会标记为"待确认"，可到异常中心修正。

## 给 Agent 的提示

- 保存调用 `manual_flow_service.quick_entry_save`，直接写入 FundEvent
- 匹配失败不阻断，标记 state="待确认"
- 收入和支出不能同时大于 0
- source 字段写入"手工录入"
