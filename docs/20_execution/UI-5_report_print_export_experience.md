# UI-5：报表打印与导出体验治理（后续规划）

> 阶段：UI-5（未开始） | 优先级：P2 | 日期：2026-05-12

## 背景

PR #33（UI-4D/4E/4F）完成了打印区域契约、全仓搜索、中文文件名等基础收尾。当前打印仍使用 `window.print()` 直接输出浏览器渲染的页面，Excel 导出为原始数据。用户期望更专业的报表输出。

## PR #33 收尾内容（已完成）

- 修复 PDF 导出前置空白页（移除 `break-inside: avoid` + `min-height: 100vh`）
- Excel 导出文件名改为中文业务名+日期区间
- 打印区域 CSS 契约（`.report-print-root` / `.report-print-root-wrapper`）

## UI-5 待办范围

### 1. 统一 PrintLayout 组件

- 专用打印容器组件，不污染正常页面 UI
- 页眉：公司名称 + 报表标题 + 日期范围
- 页脚：页码（第 X 页 / 共 Y 页）+ 打印日期
- 每页自动重复表头（`thead { display: table-header-group }` 已有基础）

### 2. 智能分页

- 按数据行数/高度自动插入分页符
- 避免合计行/小计行与数据行分离
- 跨页保持表头可见

### 3. 表格排版增强

- 表头居中对齐
- 金额列右对齐 + 千分位格式
- 合计行/小计行视觉区分
- 列宽按内容自适应

### 4. Excel 样式化导出

- 表头冻结
- 列宽自适应
- 金额列数字格式（非文本）
- 合计行加粗/背景色
- 多 sheet 支持（按单位分 sheet）

### 5. PDF 直接生成

- 评估 jsPDF / html2canvas / 后端 WeasyPrint 等方案
- PDF 输出固定 A4 排版，不依赖浏览器渲染差异
- 中文文件名 PDF 输出

## 技术方向

- 打印优先考虑 CSS `@media print` + 专用 PrintLayout 组件，不引入 PDF 生成库
- Excel 样式化可考虑 xlsx-js-style 或后端 openpyxl 模板
- PDF 直接生成作为增强选项，不阻塞 UI-5 其他项
