# 23 · API 契约

本文档定义 API 的当前约束和迁移目标。旧端点清单已移除，避免继续指导错误实现。

## 1. 统一响应格式

所有业务 API 必须返回：

```json
{ "code": 0, "message": "ok", "data": {} }
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | integer | `0` 表示成功，非零表示错误 |
| `message` | string | 中文提示 |
| `data` | object / array / null | 返回数据 |

## 2. 当前真实状态

`tools/guards/check_api_inventory.py` 当前迁移基线为 152 个端点，长期目标为 42。

这不是新的产品目标，而是为了阻止端点继续膨胀的迁移基线。后续开发不得继续新增端点来绕过旧接口缺陷；每合并或删除一批旧端点后，必须下调 guard 基线。

## 3. 目标 API 分组

API 应收敛到以下能力组：

| 分组 | 目标职责 |
| --- | --- |
| Auth | 本地登录、当前用户、密码修改 |
| Master Data | 板块、主体、银行、账户、别名、期初余额 |
| Rule Center | Parser、Rule、模板识别规则、审核记录 |
| Bank Import | 上传、匹配 Parser、预览、确认入库、批次追溯 |
| Manual Flow | 快速录入、上传、匹配规则、预览、异常维护、确认入库 |
| Base Data | `fund_events` 查询、筛选、作废、回滚 |
| Reports | Rule 执行、报表生成、下载、历史 |
| Agent | 会话、Skill、Memory、工具权限、规则创建 |
| System | 健康检查、备份、恢复、日志 |

## 4. 必须迁移的旧入口

以下入口属于迁移对象，不得作为新功能开发依据：

| 旧入口类型 | 处理方向 |
| --- | --- |
| 字段映射提交 | 迁移到 Parser 匹配、预览、确认入库 |
| 智能列映射接口 | 迁移到 Agent 创建 Parser 或 Rule 草稿 |
| 旧模板管理接口 | 迁移到 Rule Center |
| 细碎重复的主数据接口 | 合并调用方后收敛 |
| 直接报表查询入口 | 与 Rule 生成和报表历史统一 |

## 5. 新增和删除规则

- 新增端点必须先说明为什么不能复用现有资源。
- 删除端点必须先盘点前端 API client 和浏览器路径。
- 兼容转发必须有删除期限。
- API 数量基线变更必须更新本文件、guard 和验收记录。
- 路由层不得承载业务逻辑。

## 6. 错误码

| 码 | 消息 | 场景 |
| --- | --- | --- |
| `0` | `ok` | 成功 |
| `1001` | `参数缺失` | 必填字段为空 |
| `1002` | `参数格式错误` | 日期、金额、编码格式错误 |
| `1003` | `权限不足` | 未授权访问 |
| `2001` | `账户不存在` | 主数据无法匹配 |
| `2002` | `金额互斥冲突` | 收入和支出同时大于 0 |
| `2003` | `状态不允许` | 对已作废或锁定数据执行非法操作 |
| `2004` | `占位符未绑定` | Rule 未覆盖模板占位符 |
| `3001` | `Parser 生成失败` | Agent 规则草稿不合法 |
| `3002` | `样本校验未通过` | Parser 或 Rule 校验失败 |
| `3003` | `基元库调用越界` | Artifact 扫描失败 |
| `3004` | `执行超时` | 确定性执行超过限制 |
| `3005` | `Rule 绑定不完整` | 报表规则不满足模板要求 |
| `4001` | `AI Provider 连接失败` | Agent 创建规则阶段模型连接失败 |
| `4002` | `AI Provider 返回格式错误` | Agent 创建规则阶段返回不可解析 |
| `5001` | `数据库错误` | 数据库异常 |
| `5002` | `文件读写错误` | 上传、导出或备份异常 |
| `5999` | `未知错误` | 兜底 |

## 7. 验收

- `tools/guards/check_api_inventory.py` 通过。
- 端点数量不超过当前迁移基线，并持续向长期目标收敛。
- 前端 API client 不调用迁移完成后的旧入口。
- 浏览器可跑通上传、规则创建或匹配、预览、确认、入库、报表生成。
