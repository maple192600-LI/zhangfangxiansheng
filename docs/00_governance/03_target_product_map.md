# 目标产品地图

> 本文件描述产品目标是**什么能力**，不描述当前**怎么实现**。
> 当前实现状态见 [01_project_map.md](01_project_map.md)。

---

## 核心定位

账房先生是一个**本地部署的财务工作台**，服务中国中小企业和代账会计。

产品核心循环：

```
企业初始化 → 导入/录入流水 → 确认入库 → 按模板导出报表
```

---

## 多企业适配原则

**不同企业有以下差异：**

| 维度 | 差异示例 |
|------|---------|
| 单位结构 | 有的只有 1 个法人，有的有 10 个板块、50 个法人 |
| 账户体系 | 有的全有网银，有的只有手工账户和现金 |
| 银行流水格式 | 每家银行导出的 Excel 格式不同 |
| 手工录入习惯 | 有的用总表，有的逐笔录入，字段也不同 |
| 报表模板 | 现金日记账、资金日报、周报、月报，格式和占位符各不相同 |

**系统通过四层适配机制服务不同企业：**

1. **主数据**（板块/法人/账户/别名）— 每家企业自己维护
2. **ParserArtifact** — 每种银行流水格式对应一个解析器，企业可以有多种
3. **RuleArtifact** — 每个报表模板对应一条填充规则，企业可以有多套
4. **ReportTemplate** — 企业自己的报表模板文件

**禁止把某一家企业的表格写死成通用产品规则。** 所有格式适配必须通过主数据 + Artifact + Template 完成。

## 目标交付流程

### 第一步：企业初始化

用户第一次使用时，完成：

- 创建板块和法人
- 维护银行信息
- 建立账户（含账户编码、别名、期初余额）
- 配置 AI Provider（用于后续 Agent 创建解析器和规则）

### 第二步：账户数据中心

企业按自己的实际结构维护：

- 法人层级（板块 → 法人）
- 账户信息（银行账户、手工账户、现金账户）
- 账户别名（用于流水识别时命中）
- 期初余额和余额日期

### 第三步：银行流水样本上传

用户上传一份某家银行的流水 Excel，系统通过 Agent 分析样本，生成 ParserArtifact。

- Agent 识别列头、日期格式、金额列、摘要列
- 生成解析器代码
- 用户审核确认

### 第四步：ParserArtifact 审核与激活

- 用户在 AgentReview 页面查看生成的解析器
- 确认字段映射正确
- 样本校验通过
- 激活为 active 状态

后续同一银行的流水直接匹配已激活的解析器，无需再次生成。

### 第五步：手工 Excel 样本上传（如需要）

对于无网银的账户，用户上传手工 Excel 样本：

- 系统按字段方案识别列
- Agent 生成手工解析器
- 用户审核确认

### 第六步：报表模板上传

用户上传自己使用的报表模板（如现金日记账、资金日报）：

- Agent 识别模板中的占位符
- 生成 RuleArtifact（占位符 → 数据字段的映射）
- 用户审核确认

### 第七步：RuleArtifact 审核与激活

- 用户查看占位符绑定是否正确
- 确认后激活
- 后续生成报表时直接使用已激活的规则

### 第八步：日常流水导入

企业日常使用：

1. 上传银行流水 → 自动匹配 ParserArtifact → 预览 → 确认入库
2. 手工快速录入或 Excel 上传 → 预览 → 确认入库
3. 异常记录进入异常中心，用户手动修正

### 第九步：预览确认

所有导入必须经过预览：

- 展示解析结果（成功行数、异常行数）
- 用户确认后才写入 FundEvent
- 不允许静默入库

### 第十步：写入 FundEvent

确认后写入资金流水事实表，成为所有报表的数据源。

### 第十一步：按模板导出报表

- 选择报表模板（如现金日记账、资金日报）
- 选择日期范围和账户
- 系统用 RuleArtifact 填充模板
- 导出 Excel

## 目标能力清单

| 能力 | 说明 |
|------|------|
| 企业初始化向导 | 引导新用户完成首次配置 |
| 主数据管理 | 板块/法人/银行/账户/别名 CRUD |
| 银行流水导入 | 上传 → 解析 → 预览 → 入库 |
| 手工流水双轨 | 快速录入 + Excel 多主体上传 |
| 异常处理 | 待确认/异常记录的查看和修正 |
| 报表生成 | 按模板生成，支持多种报表类型 |
| 报表导出 | 导出 Excel，金额与页面一致 |
| Agent 创建解析器 | 从样本生成 ParserArtifact |
| Agent 创建规则 | 从模板生成 RuleArtifact |
| 产物审核 | 用户审核并激活 Parser/Rule |
| 备份与恢复 | SQLite 全库备份 + 恢复 |
| 操作日志 | 关键操作留痕 |

---

## 不属于目标能力

- 发票 OCR 正式流程
- 费用审批
- 合同管理
- 多人协作
- 集中部署
- 自定义字段映射 UI
