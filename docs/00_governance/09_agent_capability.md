# 09 · Agent 能力契约

本文档定义账房先生的 Agent、Skill、Memory、Parser、Rule 能力边界。

## 1. Agent 定位

Agent 负责把用户的样本、模板、口头规则和纠错意见转化为可复用、可审核、可追溯的规则资产。

Agent 负责：

- 和用户讨论业务含义。
- 分析首次上传的银行流水或报表模板。
- 创建 Parser 或 Rule 草稿。
- 解释解析结果和异常原因。
- 根据用户确认沉淀 Skill 和 Memory。
- 维护已有规则的修正记录。

Agent 不负责：

- 在日常导入执行阶段自由决定字段映射。
- 在报表生成阶段调用模型取数或填表。
- 绕过用户确认写入正式基础数据。
- 让用户写 JSON、正则、SQL 或字段映射。

## 2. 产品主链路

```text
首次样本或模板
  -> Agent 讨论和分析
  -> 创建 Parser 或 Rule 草稿
  -> 样本校验
  -> 用户审核
  -> 规则中心保存

日常工作
  -> 上传文件
  -> 匹配已审核 Parser 或 Rule
  -> 确定性执行
  -> 预览
  -> 用户确认
  -> fund_events 或报表结果
```

关键原则：Agent 参与规则创建，确定性代码执行已审核规则。

## 3. 核心概念

| 概念 | 含义 | 长期资产 |
| --- | --- | --- |
| Agent | 会讨论、学习、建规则的助手 | 会话、设置、权限、记忆 |
| Skill | Agent 做某类工作的操作说明 | `SKILL.md` |
| Memory | 可审计、可删除、已脱敏的长期经验 | 用户偏好、业务事实、规则经验、纠错记录 |
| Parser | 把流水或手工表转成标准资金事件的规则 | Parser artifact |
| Rule | 从基础数据表生成报表或填模板的规则 | Rule artifact |
| Artifact | 已审核可执行规则资产 | 代码、参数、校验、审批记录 |

## 4. Agent 运行时

Agent 运行时可以调用模型和工具，因为它处在规则创建、讨论和维护阶段。

运行时必须具备：

- 系统提示词组装。
- Memory 注入和清洗。
- Skill 匹配和加载。
- 工具权限控制。
- 文件解析。
- 用户确认。
- 会话持久化。
- 调用审计。

涉及用户真实数据时，Agent 只能接触完成任务所需的最小样本。敏感金额、账号、对方信息应按隐私设置脱敏。

## 5. 确定性执行边界

以下阶段不得调用 LLM：

- 银行流水 Parser 执行。
- 手工流水 Parser 执行。
- 写入 `fund_events`。
- 报表 Rule 执行。
- 余额滚动计算。
- 汇总、筛选、导出。

如果需要模型参与，必须回到“规则创建或规则修正”流程，并重新经过用户审核。

## 6. Skill 标准

Skill 必须使用 `SKILL.md`，并至少说明：

- 触发条件。
- 输入。
- 输出。
- 允许工具。
- 工作流程。
- 校验标准。
- 禁止事项。

Skill 是 Agent 的工作说明，不是日常执行入口。日常执行入口是已审核 Parser 或 Rule。

## 7. Memory 标准

Memory 分为四类：

| 类型 | 示例 |
| --- | --- |
| 用户偏好 | 喜欢日报先看余额，再看收支 |
| 业务事实 | 某主体常用简称、账户归属关系 |
| 规则经验 | 中国银行网银流水常见表头别名 |
| 纠错记录 | 用户指出某列不应作为摘要 |

Memory 必须：

- 可查看。
- 可脱敏。
- 可删除。
- 可追溯来源。
- 不隐藏保存敏感流水明细。

## 8. Parser 创建要求

Agent 创建 Parser 时，必须输出：

- 适用银行、渠道、文件类型和格式特征。
- 字段提取规则。
- 主体和账户匹配线索。
- 样本校验结果。
- 异常路由标准。
- 用户确认记录。

Parser 不得绑定单个账户。银行流水 Parser 应面向银行和格式复用。

## 9. Rule 创建要求

Agent 创建 Rule 时，必须输出：

- 适用报表或模板。
- 所需基础字段。
- 取数范围。
- 汇总和排序规则。
- 模板占位符绑定。
- 样本生成结果。
- 用户确认记录。

Rule 执行时只能从 `fund_events` 和必要主数据取数。

## 10. 规则中心

规则中心统一管理：

- 银行 Parser。
- 手工 Parser。
- 报表 Rule。
- 模板识别规则。
- 规则审核记录。
- 样本校验记录。
- 规则变更历史。

规则中心展示的是可复用资产，不是一次性聊天结果。

## 11. 硬边界

Agent 和规则系统禁止：

- 绕过审核执行未批准 artifact。
- 在确定性执行阶段调用 LLM。
- 把用户原始明细写入隐藏 Memory。
- 让用户直接维护 JSON、正则、SQL 或字段映射。
- 用新文件或新 API 绕过旧缺陷形成平行实现。
- 把参考项目的 Agent 机制当作账房先生权威。

## 12. 验收标准

- 首次中国银行样本可通过 Agent 创建 Parser。
- 不同公司、不同账户的同格式中国银行流水能复用同一 Parser。
- 报表模板首次上传可创建 Rule。
- 报表生成只读取 `fund_events` 和 Rule。
- Agent 创建规则后能沉淀字段别名、银行别名、样本校验和用户确认意见。
- Memory 可查看、脱敏、删除。
