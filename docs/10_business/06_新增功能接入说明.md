# 新增功能接入说明

> 给未来开发者的检查清单：新增一个业务功能时需要做什么。

---

## 新增报表页面

### 前端步骤

1. 在 `frontend/src/views/` 创建 Vue 文件
2. 使用 `report-print-root-wrapper` + `report-print-root` 包裹打印区域
3. 筛选栏使用 `filters-bar` 样式，操作按钮放在 `btn-row` 中
4. 导入 `useReportPrint` 实现打印功能
5. 导入 `useTemplateColumns` 支持模板列配置（增强层，缺失时降级到默认列）
6. 导入 `getReportFilename`（from `@/utils/reportFilename`）统一导出文件名
7. 在 `reportFilename.js` 的 `REPORT_NAMES` 中注册新报表的中文名
8. 在 `frontend/src/router/` 注册路由

### 后端步骤

1. 在 `backend/services/report_service.py` 添加查询函数，只查询 `state='正常'` 的 FundEvent
2. 在 `backend/api/reports.py` 添加路由
3. 在 `frontend/src/api/report.js` 添加前端 API 函数
4. 导出接口如果需要，在 `backend/services/export_service.py` 添加对应逻辑

### 检查清单

- [ ] 有默认列配置（不依赖模板存在）
- [ ] 日期筛选为空时有合理 fallback
- [ ] 导出文件名使用 `getReportFilename`
- [ ] 打印区域正确包裹
- [ ] 分页（数据量大时）
- [ ] 金额列右对齐，使用 `fmtAmt` 格式化

---

## 新增导入方式

### 必须遵守

1. 所有导入必须创建 ImportBatch 记录
2. 所有导入必须经过预览确认，不允许静默入库
3. 入库数据必须符合 CANONICAL_12 格式
4. 收入支出互斥约束必须遵守
5. 匹配不到法人/账户时标记为"待确认"

### 步骤

1. 在 `manual_flow_service.py` 或新建 service 文件实现业务逻辑
2. 创建 ImportBatch（`source_type` 用新类型）
3. 解析后写入 FundEvent
4. 在异常中心能处理新类型的异常

---

## 新增主数据类型

1. 在 `backend/db/tables.py` 创建 ORM 类
2. 在 `backend/api/master_data.py` 添加 CRUD 路由
3. 在 `frontend/src/api/master.js` 添加 API 函数
4. 在「账户管理」页面添加管理入口

---

## 通用注意事项

- 增强层不阻断核心功能：模板、主题缺失时必须降级到默认行为
- 流水数据未来会很大：查询必须带日期范围筛选，不做全量加载
- 报表生成禁止调 LLM（§C8 约束）
- 中文 UI：所有面向用户的信息用中文
