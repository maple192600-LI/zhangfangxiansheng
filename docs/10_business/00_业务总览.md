# 业务总览

> 本文档面向业务人员，用业务语言描述"账房先生是什么、解决什么问题"。

## 产品定位

账房先生是一个**本地部署的财务工作台**，服务中国中小企业的代账会计和财务人员。

核心循环：

```
企业初始化 → 导入/录入流水 → 确认入库 → 按模板查看和导出报表
```

## 解决的问题

| 痛点 | 账房先生怎么解决 |
|------|-----------------|
| 多家银行流水格式不统一 | 每种银行格式生成一个解析器，后续自动匹配 |
| 手工录入容易出错、格式混乱 | 提供结构化的快速录入表格，自动匹配单位和账户 |
| Excel 报表手工拼、容易漏行 | 上传模板 → 绑定规则 → 自动从流水数据填充 |
| 多法人、多账户汇总复杂 | 主数据统一管理，报表按单位汇总、小计、合计 |
| 异常流水散落各处 | 异常中心集中处理待确认和异常记录 |

## 使用角色

- **财务会计**：日常录入流水、查看报表、导出 Excel
- **系统管理员**（通常就是会计本人）：初始化企业信息、配置 AI 智能体

## 13 个业务流程一览

| 编号 | 流程 | 解决什么问题 | 当前状态 |
|------|------|-------------|---------|
| 1 | 企业初始化 | 第一次使用时搭建组织结构 | 可用 |
| 2 | 主数据维护 | 持续管理板块、法人、银行、账户 | 可用 |
| 3 | 手工快速录入 | 没有网银时逐笔录入流水 | 可用 |
| 4 | 手工 Excel 导入 | 有现成 Excel 表格时批量导入 | 暂不可用 |
| 5 | 银行流水导入 | 从网银下载的流水文件自动解析 | 暂不可用 |
| 6 | 预览确认 | 入库前看一眼解析结果，确认无误再提交 | 部分可用 |
| 7 | 标准资金流水入库 | 确认后的流水写入统一事实表，成为报表数据源 | 部分可用 |
| 8 | 异常处理 | 处理匹配失败、字段缺失的异常流水 | 可用 |
| 9 | 报表查询 | 按日期、单位查看日报/日记账/余额表等 | 可用 |
| 10 | 报表导出 | 导出 Excel 文件，中文文件名 | 可用 |
| 11 | 报表模板维护 | 上传和管理报表的列配置、布局 | 可用 |
| 12 | Agent 生成解析器 | AI 自动分析银行流水样本，生成解析规则 | 待激活 |
| 13 | Agent 生成报表规则 | AI 自动分析报表模板，生成填充规则 | 待激活 |

## 当前可用主线

**手工快速录入 → 报表查询/导出** 是当前完整可用的主线。

银行导入和手工 Excel 导入两条链路，因为解析器运行时（`run_parser`）尚未实现，目前无法走完全程。这两个能力将在后续阶段逐步激活。

## 数据量预估

- 每日流水：几十到几百条
- 月度累计：几百到几千条
- 年度累计：几千到几万条
- 多法人多账户叠加后可能达到十万级别

因此报表查询和导出都按日期范围筛选，不做全量加载。
