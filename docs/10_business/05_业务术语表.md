# 业务术语表

> 统一项目中的业务术语，避免歧义。

---

## 组织结构

| 术语 | 含义 | 在系统中对应 |
|------|------|-------------|
| 板块 | 企业的组织层级，如"集团本部"、"华东事业部" | divisions 表 |
| 法人/法人实体 | 独立法人单位，有独立的银行账户 | entities 表 |
| 法人编码 | 法人的唯一标识，如 E001 | entity_code 字段 |
| 法人全称 | 法人的注册名称，如"XX建设有限公司" | name 字段 |
| 法人简称 | 日常使用的短名称，如"XX建设" | short_name 字段 |

## 金融

| 术语 | 含义 | 在系统中对应 |
|------|------|-------------|
| 账户 | 一个银行账户或资金载体 | accounts 表 |
| 账户编码 | 账户的唯一标识，如 A001 | account_code 字段 |
| 账户别名 | 账户的显示名称，如"招行基本户" | account_alias 字段 |
| 期初余额 | 查询期间开始时的账户余额 | 计算值：initial_balance + 历史净变动 |
| 期末余额 | 查询期间结束时的账户余额 | 计算值：期初 + 期间收入 - 期间支出 |
| 滚动余额 | 某条流水记录时点的累计余额 | rolling_balance 字段 |
| 收入/收入金额 | 资金流入 | amount_in 字段 |
| 支出/支出金额 | 资金流出 | amount_out 字段 |
| 净变动 | 期间收入 - 期间支出 | 计算值 |

## 流水

| 术语 | 含义 | 在系统中对应 |
|------|------|-------------|
| 流水 | 一笔资金收支记录 | fund_events 表的每一行 |
| 标准流水 | 符合 CANONICAL_12 格式的标准化流水记录 | FundEvent 的 12 个核心字段 |
| 摘要 | 流水的业务说明，如"收到工程款" | summary 字段 |
| 对方 | 交易的对手方名称 | counterparty 字段 |
| 批次 | 一次导入操作（文件上传或快速录入）产生的一批流水 | import_batches 表 |
| 网银导入 | 从银行下载的交易明细文件导入 | source = '网银导入' |
| 手工录入 | 通过快速录入界面逐笔输入 | source = '手工录入' |

## 报表

| 术语 | 含义 |
|------|------|
| 资金日报 | 按法人汇总的每日资金状况 |
| 现金日记账 | 按账户分块的每日收支明细 |
| 账户余额表 | 按法人→账户层级的余额汇总 |
| 基础数据表 | 所有流水记录的原始视图 |
| 月度对账表 | 月末账户余额核对表 |
| 小计行 | 报表中按法人汇总的行 |

## 系统概念

| 术语 | 含义 | 注意事项 |
|------|------|---------|
| 解析器/ParserArtifact | AI 生成的、用于解析银行流水格式的代码产物 | 状态：draft → active → retired |
| 规则/RuleArtifact | AI 生成的、用于填充报表模板的绑定规则 | 状态：draft → active → retired |
| 模板/ReportTemplate | 报表的列配置和 Excel 布局定义 | 每种报表类型可有多个模板，一个为默认 |
| 智能体/Agent | 通用 AI 助手，通过聊天生成解析器和规则 | 不是独立的"领域 Agent" |
| 确定性执行 | 解析器和规则执行时不调用 LLM，纯代码运行 | 核心约束：§C8 |
| CANONICAL_12 | 标准流水的 12 个必填字段格式 | 所有来源的流水最终都统一为此格式 |
