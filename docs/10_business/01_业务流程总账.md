# 业务流程总账

> 每条流程按"解决什么问题、谁使用、正常流程、异常情况、影响哪些报表、验收标准"描述。

---

## 1. 企业初始化

**解决什么问题：** 第一次使用时，把企业的组织结构告诉系统。

**谁使用：** 系统管理员（通常就是会计本人）。

**正常流程：**
1. 在「账户管理」页面创建板块（如"集团本部"）
2. 在板块下创建法人实体（如"XX建设有限公司"），填写简称和编码
3. 创建银行信息（如"招商银行"、"建设银行"）
4. 在法人下创建账户，填写账户编号、别名、开户行、期初余额、余额日期
5. 配置 AI Provider（如选智谱清言、填 API Key），为后续 AI 解析做准备

**异常情况：**
- 法人编码重复 → 系统拒绝创建，提示修改
- 账户编号重复 → 系统拒绝创建
- 期初余额未填 → 默认为 0

**影响报表：** 所有报表依赖法人、账户数据。日报按法人汇总，日记账按账户分块。

**验收标准：** 创建至少 1 个法人、1 个银行、1 个账户后，资金日报能正常显示该法人的数据。

---

## 2. 主数据维护

**解决什么问题：** 企业组织结构会变化（新增法人、变更银行、调整账户），需要持续维护。

**谁使用：** 财务会计。

**正常流程：**
1. 新增法人：在「账户管理」页面新增，填写编码、全称、简称
2. 新增账户：选择法人 → 选择银行 → 填写账户信息
3. 设置账户别名：一个账户可以有多个别名，用于流水匹配时的识别
4. 停用/启用：停用的账户不参与报表计算

**异常情况：**
- 已有流水的账户删除 → 系统不允许删除，只能停用
- 账户编码变更 → 不允许修改编码，只能新增账户

**影响报表：** 停用的账户不出现在日报和余额表中。账户的 `include_in_daily_report` 开关控制是否参与日报计算。

**验收标准：** 新增账户后，当天的流水录入能匹配到该账户；停用后报表不再显示。

## 3. 手工快速录入

**解决什么问题：** 没有网银下载的流水（如现金收付、内部转账），需要手工逐笔录入。

**谁使用：** 财务会计。

**正常流程：**
1. 在「手工流水录入」页面，选择"快速录入"模式
2. 系统展示可编辑表格，列包括：单位简称、账户编号、日期、摘要、对方、收入金额、支出金额等
3. 点击"添加行"，逐行填写
4. 点击"保存到暂存区"
5. 系统自动匹配单位和账户：输入法人简称或编码 → 自动关联；输入账户编号或别名 → 自动关联
6. 保存后自动跳转到预览页面

**异常情况：**
- 法人简称输入错误、系统无法匹配 → 该条流水标记为"待确认"，进入异常中心
- 账户编号输入错误 → 同上
- 日期格式不对 → 同上
- 收入和支出同时填了非零值 → 系统校验拒绝，标记为异常
- 收入和支出都是 0 → 标记为异常

**影响报表：** 保存成功的数据立即写入流水事实表（FundEvent），可以在所有报表中查到。

**验收标准：** 录入 3 条正常流水后，资金日报和现金日记账能正确显示。

---

## 4. 手工 Excel 导入

**解决什么问题：** 手头有现成的 Excel 表格（如从其他系统导出的），需要批量导入。

**谁使用：** 财务会计。

**正常流程：**
1. 在「手工流水录入」页面，先下载录入模板（Excel 文件，含表头和字段说明）
2. 按模板格式填写数据
3. 选择"Excel 上传"模式，拖拽或选择文件上传
4. 系统解析文件，展示预览
5. 确认后提交入库

**当前状态：** ⚠️ **暂不可用**。文件可以上传，但解析和入库依赖解析器运行时（`run_parser`），该能力尚未实现。当前上传 Excel 时会提示用户使用快速录入。

**异常情况：**
- 文件格式不支持（非 xls/xlsx/csv）→ 提示错误
- 表头无法识别 → 需要 AI 辅助识别或手工选择列映射

**影响报表：** 与手工快速录入相同，成功入库后立即可查。

**验收标准：** 上传一个包含 5 行数据的 Excel 文件，预览显示 5 条解析结果，确认后资金日报能正确汇总。

## 5. 银行流水导入

**解决什么问题：** 从网银下载的交易明细（Excel 格式），每家银行格式不同，需要自动解析。

**谁使用：** 财务会计。

**正常流程：**
1. 在「银行流水导入」页面，拖拽或选择从网银下载的 Excel 文件
2. 系统上传文件，自动检测文件格式和表头
3. 自动匹配已审核的解析器（ParserArtifact）：每个银行格式对应一个解析器
4. 如果匹配到解析器 → 自动进入预览
5. 预览显示解析结果：有效行数、异常行数
6. 确认提交 → 数据写入流水事实表
7. 显示导入完成结果

**当前状态：** ⚠️ **暂不可用**。文件上传步骤可以完成（能识别格式和表头），但预览和提交依赖解析器运行时（`run_parser`），该能力尚未实现。如果当前没有已审核的解析器，页面会提示"请先创建解析规则"。

**异常情况：**
- 无匹配的解析器 → 提示需要先创建解析规则
- 解析结果中有异常行（字段缺失、格式不对）→ 分开展示，正常行和异常行分开计数
- 重复导入同一文件 → 允许，会创建新批次，旧数据不会被覆盖（但同一批次的旧 FundEvent 会被清除后重建）

**影响报表：** 入库后的数据立即可在所有报表中查看。

**验收标准：** 上传某银行的流水文件，预览显示正确的日期、摘要、金额列，确认后资金日报和现金日记账能正确汇总。

## 6. 预览确认

**解决什么问题：** 所有导入必须经过人工确认，不允许静默入库。

**谁使用：** 财务会计。

**正常流程：**
1. 上传文件或保存手工录入后，系统展示预览结果
2. 预览显示：总行数、有效行数、异常行数
3. 有效行展示解析后的标准化字段（日期、单位、账户、摘要、金额等）
4. 异常行单独展示，并标注异常原因（如"ENTITY_MATCH_FAILED"、"DATE_INVALID"）
5. 用户确认 → 正常行写入流水事实表，异常行以"待确认"状态写入

**当前状态：** 手工快速录入的预览可用。银行导入和手工 Excel 的预览依赖解析器运行时。

**异常情况：**
- 全部行都是异常 → 仍然允许提交，所有行以"待确认"状态入库，用户可到异常中心逐一修正
- 预览超时 → 提示用户检查文件大小

**验收标准：** 上传包含正常行和异常行的数据，预览正确区分两种行，确认后正常行状态为"正常"，异常行状态为"待确认"。

## 7. 标准资金流水入库

**解决什么问题：** 所有来源的流水最终统一成标准格式，成为报表的唯一数据源。

**谁使用：** 系统内部，用户通过确认操作间接触发。

**标准格式（CANONICAL_12）：**

| 字段 | 含义 | 示例 |
|------|------|------|
| business_date | 业务日期 | 2026-05-12 |
| entity_code | 法人编码 | E001 |
| entity_name | 法人名称 | XX建设有限公司 |
| account_code | 账户编码 | A001 |
| account_name | 账户名称 | 招行基本户 |
| summary | 摘要 | 收到工程款 |
| counterparty | 对方 | XX公司 |
| amount_in | 收入金额 | 100000.00 |
| amount_out | 支出金额 | 0 |
| rolling_balance | 滚动余额 | 500000.00 |
| state | 状态 | 正常 |
| source | 来源 | 网银导入/手工录入/现金录入 |

**关键约束：**
- 收入和支出不能同时大于 0（互斥约束）
- 状态只有四种：正常、待确认、异常、已作废
- 来源只有五种：网银导入、手工录入、现金录入、票据录入、财务公司单据

**验收标准：** 手工录入一条流水后，在基础数据表中能查到完整的 12 个字段。

## 8. 异常处理

**解决什么问题：** 导入时无法自动匹配法人/账户、日期格式不对、金额异常的流水，需要人工修正。

**谁使用：** 财务会计。

**正常流程：**
1. 在「异常中心」页面查看所有待确认和异常状态的流水
2. 可以按状态筛选（待确认/异常）
3. 点击"修正"：修改日期、摘要、对方、金额、余额等字段
4. 点击"保存并标记正常"：修正后的流水状态变为"正常"，立即可在报表中看到
5. 点击"作废"：输入作废原因，流水状态变为"已作废"，不再参与报表计算

**异常情况：**
- 修正后仍然匹配不到法人 → 可以直接修改法人名称字段，标记正常后手动修正
- 批量异常 → 目前需要逐条修正

**影响报表：** 标记正常后立即生效。已作废的流水不参与任何报表计算。

**验收标准：** 异常中心有 1 条待确认流水，修正后标记正常，该流水在资金日报和基础数据表中都能看到。

---

## 9. 报表查询

**解决什么问题：** 按日期范围、单位查看各种报表，了解资金状况。

**谁使用：** 财务会计。

**可用报表：**

| 报表 | 展示内容 | 筛选条件 |
|------|---------|---------|
| 资金日报 | 按法人汇总：期初/收入/支出/净变动/期末 | 日期范围 + 单位 |
| 账户余额表 | 按法人→账户层级：期初/本期收入/本期支出/期末 | 日期范围 + 单位 |
| 现金日记账 | 按账户分块，按日展示：上日余额/收入/支出/本日余额 | 日期范围 + 账户 |
| 收入明细 | 每笔收入流水的详细信息 | 日期范围 + 单位 |
| 支出明细 | 每笔支出流水的详细信息 | 日期范围 + 单位 |
| 基础数据表 | 所有流水的原始记录 | 日期范围 + 单位 + 方向 + 关键词 |
| 资金余额表 | 主要银行存款账户的余额 | 日期范围 + 单位 |
| 月度对账表 | 指定年月的账户余额 | 年 + 月 + 单位 |
| 周报/月报/年报 | 按周/月/年汇总 | 日期范围或年月 + 单位 |

**正常流程：**
1. 选择日期范围和筛选条件
2. 点击"生成报表"
3. 系统从流水事实表（FundEvent）实时计算，展示结果
4. 如果有已上传的报表模板，按模板的列配置和布局展示

**异常情况：**
- 无数据 → 显示"暂无数据"提示
- 日期范围太大 → 可以查询，但分页展示（基础数据表、收支明细有分页）

**验收标准：** 录入几笔流水后，选择对应日期范围，日报、余额表、日记账数据一致（合计金额吻合）。

## 10. 报表导出

**解决什么问题：** 需要把报表发给老板、存档、或导入其他系统时，导出 Excel 文件。

**谁使用：** 财务会计。

**正常流程：**
1. 在任意报表页面，设置好筛选条件
2. 点击"导出"
3. 系统生成 Excel 文件，浏览器自动下载
4. 文件名格式：`报表名_开始日期_结束日期.xlsx`

**文件名规则：**

| 场景 | 文件名示例 |
|------|-----------|
| 有开始和结束日期 | `资金日报_2026-05-01_2026-05-12.xlsx` |
| 只有开始日期 | `资金日报_2026-05-01起.xlsx` |
| 只有结束日期 | `资金日报_截至2026-05-12.xlsx` |
| 都没有 | `资金日报_全部.xlsx` |
| 月报 | `月报_2026年5月.xlsx` |
| 年报 | `年报_2026年.xlsx` |

**异常情况：**
- 导出失败 → 弹出错误提示
- 日期为空 → 文件名自动用"全部"替代

**验收标准：** 导出资金日报后，Excel 内容与页面显示的数据行数和金额完全一致。

## 11. 报表模板维护

**解决什么问题：** 不同企业、不同报表的列配置不同。模板让报表按企业需要的格式展示。

**谁使用：** 财务会计。

**正常流程：**
1. 在「报表模板管理」页面，左侧选择报表类型（如"日报"、"日记账"）
2. 点击"新建模板"：手动配置列（字段名、表头名、宽度、对齐方式）
3. 或者点击"上传 Excel 解析"：上传一个 Excel 文件，系统自动识别表头列配置
4. 可以设为默认模板：设为默认后，对应报表页面优先使用该模板的列配置展示
5. 编辑、删除已有模板

**异常情况：**
- 上传的 Excel 无有效表头 → 解析失败
- 同一报表类型无默认模板 → 报表页面使用内置默认列配置

**影响报表：** 有模板时按模板展示，无模板时用系统默认列。模板缺失不阻断报表查询。

**验收标准：** 上传一个包含 8 列的 Excel 模板，设为默认后，对应报表页面展示的列配置与模板一致。

## 12. Agent 生成解析器

**解决什么问题：** 每家银行流水格式不同，人工编写解析规则成本高。AI 自动分析样本，生成解析代码。

**谁使用：** 系统管理员（需要 AI 配置）。

**正常流程：**
1. 在 AI 智能体中创建财务助手，关联已配置的 AI Provider
2. 向智能体上传一份银行流水样本文件
3. 智能体分析样本，生成 ParserArtifact（解析器产物），包含：
   - 识别了哪些列（日期、摘要、金额等）
   - 解析代码（Python 脚本）
   - 置信度评估
4. 在「Agent 审核」页面查看生成的解析器
5. 确认无误后，审批激活
6. 后续同一银行的流水自动匹配该解析器

**当前状态：** ⚠️ Agent 可以生成解析器草稿，但 `run_parser` 执行器尚未实现。即使审批激活，实际导入时仍无法执行。

**异常情况：**
- AI 配置无效 → 智能体无法调用，返回错误
- 解析器代码有安全问题 → 系统有 AST 白名单校验，禁止导入非法模块

**验收标准：** 上传某银行样本，Agent 生成解析器，审批激活后，上传同一银行的另一份流水能正确解析入库。

## 13. Agent 生成报表规则

**解决什么问题：** 报表模板中的占位符（如 `{{报表标题}}`、`{{期初余额}}`）需要映射到数据字段，AI 自动完成绑定。

**谁使用：** 系统管理员。

**正常流程：**
1. 在「报表模板管理」页面上传 Excel 模板
2. 系统自动识别模板中的占位符（Stage A）
3. 智能体分析占位符与数据字段的对应关系（Stage B）
4. 生成 RuleArtifact（规则产物），包含占位符 → 字段的绑定关系
5. 用户在审核页面确认绑定是否正确
6. 激活后，对应报表可以使用规则填充模板

**当前状态：** Stage A（占位符识别）可用。Stage B（AI 语义映射）为简单字符串匹配，完整能力待实现。`run_rule` 执行器尚未实现。

**异常情况：**
- 模板无占位符 → 无法生成规则
- 占位符含义不明确 → AI 映射可能不准确，需要人工确认

**验收标准：** 上传包含占位符的模板，Agent 生成规则并激活后，生成报表时模板中的占位符被正确的数据替换。
