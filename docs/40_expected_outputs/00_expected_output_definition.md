# 00｜期望输出定义与使用规则

## 一、什么叫“期望输出”

在这个项目里，期望输出不是“希望系统最后大概长成什么样”，也不是“说明性效果描述”。

期望输出指的是：

**针对某个输入、某个模块、某个页面、某个接口、某个流程，系统必须稳定产出的可检查结果。**

它必须满足四个条件：

第一，可被验证。
第二，可被复现。
第三，可被对照。
第四，可作为 Claude Code / Codex 的完成依据。

所以，期望输出本质上是开发和测试的共同中间层。

它把“业务意图”翻译成“可以检查的结果”。

---

## 二、期望输出和普通需求描述的区别

普通需求描述会说：

- 首页要像工作总控台
- 手工流水要支持快速录入
- 上传结果预览要能看异常

这些说法能帮 AI 理解方向，但不能直接判断是否做完。

期望输出必须写成：

- 首页默认进入“工作总览”标签页，并显示待导入、待确认、待生成三类待办数量
- 手工流水页支持“系统内快速录入”和“Excel 多主体总表上传”两种模式
- 上传结果预览页必须输出待处理项数量、已处理项数量、重复疑似数量、当前版本号

前者是方向。
后者才是可执行结果。

---

## 三、哪些地方必须写期望输出

以下内容必须写期望输出，不能只写说明文字：

### 1. 页面

凡是用户真正会操作和查看的页面，都必须写期望输出。

包括：

- 首页总控台
- 网银导入页
- 手工流水页
- 手动维护页
- 上传结果预览页
- 基础数据表页
- 资金日报表页
- 导出打印页
- 备份恢复页

### 2. 接口

所有会被前端调用的接口，都必须写期望输出。

包括：

- 创建主数据
- 上传银行流水
- 上传手工总表
- 解析预览
- 写入基础交易
- 重算日报
- 导出 Excel
- 批次回滚
- 备份恢复

### 3. 数据表

所有关键表都必须写期望输出。

包括：

- accounts
- fund_events
- parser_templates
- daily_report_runs
- operation_logs
- custom_fields
- custom_field_values
- ai_configs
- agent_configs

### 4. 流程

所有跨页面、跨服务的主流程都必须写期望输出。

包括：

- 银行流水上传到基础数据表
- 手工流水上传到基础数据表
- 基础数据表到日报生成
- 日报生成到导出打印
- 导错批次后的回滚重算
- 历史区间补做

### 5. 测试样本

所有真实样本和脱敏样本都必须绑定期望输出。

否则 Claude Code 只能把流程做成“看起来能跑”，无法验证结果是否真的对。

## 四、期望输出的常见类型

### 1. 页面输出

指页面上最终应该出现什么。

例子：

- 首页默认显示“工作总览”
- 手工流水页顶部显示模式切换：快速录入 / 模板上传
- 上传结果预览页显示待处理项、已处理项、重复疑似、版本号

### 2. 数据输出

指数据库里最终应该写入什么。

例子：

- 手工多主体总表上传后，生成多条标准交易记录
- 每条记录都有 batch_id、source、raw_data
- 不能唯一归属的记录不进入正式交易表，而进入异常待处理状态

### 3. 计算输出

指系统对输入做完规则和计算后，应该得到什么结果。

例子：

- 区间日报的期初余额从上一有效余额承接
- 区间内收入、支出、期末余额计算正确
- 当天无发生额时，不强制生成日报，但下次能正确承接

### 4. 文件输出

指系统应该生成哪些文件。

例子：

- 导出 Excel 文件
- 打印版日报
- 数据备份 zip
- 批量日志导出文件

### 5. 交付输出

指一个阶段做完后必须能交出的产物。

例子：

- 可运行代码
- 初始化数据库脚本
- 前后端依赖说明
- 基础样本测试报告
- V1 交付清单

---

## 五、期望输出的标准写法

每一个期望输出建议固定使用下面这个结构：

### 输出名称

### 所属模块

### 输入前提

### 触发动作

### 期望页面结果

### 期望数据结果

### 期望异常结果

### 验证方式

### 判定为完成的标准

## 六、示例

### 输出名称
手工多主体总表上传预览结果

### 所属模块
资金板块 / 工作台 / 手工流水

### 输入前提
- 用户已存在合法法人、账户、资金载体主数据
- 用户上传一份多主体手工总表 xlsx 文件
- 文件包含合法核心字段

### 触发动作
点击“上传模板”并确认解析

### 期望页面结果
- 页面显示上传成功
- 页面跳转或进入上传结果预览
- 显示待处理项数量、已处理项数量、重复疑似数量、当前版本号

### 期望数据结果
- 系统生成一批手工批次记录
- 解析成功的记录形成候选标准资金事件
- 记录保留 raw_data

### 期望异常结果
- 缺少法人识别键的记录进入异常待确认
- 账户不能唯一命中的记录进入异常池
- 不允许静默自动入正式日报

### 验证方式
- 检查页面
- 检查数据库批次记录
- 检查异常池数量

### 判定为完成的标准
- 页面状态与数据库结果一致
- 可人工复核
- 可继续进入基础数据表

---

## 七、结论

期望输出不是“补充说明”。

它是连接需求、开发、测试、验收的核心文档层。

没有期望输出，AI coding 工具只能“尽量理解”。

有了期望输出，它才知道“做到什么算对”。
