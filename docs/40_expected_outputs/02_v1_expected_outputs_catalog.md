# 02｜V1 期望输出总目录

## 一、总说明

以下内容是到目前为止，本项目第一阶段已经明确设计出来的“期望输出”。

它们按模块列出，不再写成泛泛说明。

每一项都可以被拿去做：

- 开发完成判断
- 页面完成判断
- 接口完成判断
- 测试结果对照
- 验收依据

---

## 二、首页总控台期望输出

### 输出 01：默认进入工作总览

输入前提：用户成功打开系统首页。

期望页面结果：
- 首页位于左侧独立置顶
- 默认进入“工作总览”标签
- 展示今日工作总览、今日处理进度、系统提醒、待办追踪、快捷入口

期望数据结果：
- 待导入数量来自待处理银行流水批次
- 待确认数量来自异常池 / 待确认池
- 待生成数量来自基础数据表和日报状态

判定标准：
- 页面不是静态卡片页，而是工作总控台
- 页面上的待办数量与底层状态一致

### 输出 02：首页待办追踪可反向引导到工作台

期望页面结果：
- 待导入项跳到网银导入
- 待确认项跳到手动维护
- 待生成基础数据表跳到上传结果预览
- 待生成日报跳到资金日报表

判定标准：
- 首页不是信息孤岛，能作为工作流入口

## 四、银行流水导入期望输出

### 输出 04：银行流水可上传并形成解析预览

输入前提：用户上传 xlsx/xls/csv。

期望页面结果：
- 显示文件名、来源银行、状态
- 命中已有模板时，显示解析预览
- 未命中模板时，进入首次识别流程

期望数据结果：
- 系统生成 batch_id
- 每条候选记录保留 raw_data
- 不直接写成日报结果

判定标准：
- 预览成功且未静默自动入正式日报

### 输出 05：新模板可首次识别并保存

期望页面结果：
- 显示建议字段映射
- 用户可确认后保存模板

期望数据结果：
- parser_templates 成功新增记录
- 下次同类文件可复用模板

## 五、手工流水期望输出

### 输出 06：手工流水支持双轨机制

期望页面结果：
- 手工流水页提供两种模式：快速录入 / 模板上传
- 快速录入适合少量补录
- 模板上传适合多主体总表批量录入

判定标准：
- 手工流水不再只有一种低效表单流

### 输出 07：快速录入支持直接进入待计算池

期望页面结果：
- 用户可直接录入少量记录
- 可保存到待计算池
- 可保存并立即生成

期望数据结果：
- 记录带 source=manual
- 记录进入待处理批次

### 输出 08：多主体手工总表上传可形成批次预览

输入前提：上传合法手工总表模板。

期望页面结果：
- 显示待处理项数量、已处理项数量、重复疑似数量、当前版本号
- 支持跳转手动维护

期望数据结果：
- 解析成功的记录进入候选标准资金事件
- 缺少核心识别字段的记录进入异常池

判定标准：
- 一份手工总表可以覆盖多个法人和多个资金载体
- 不能唯一归属的记录不能静默写入正式结果

- 这些视图均来自统一基础数据和日报计算口径
