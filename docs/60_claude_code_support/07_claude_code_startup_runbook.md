# Claude Code 开工运行手册

## 一、作用

这份文档不是需求文档。

它只解决一件事：当 Claude Code 正式开工做 V1 时，第一天、第一轮、第一周该怎么按顺序启动，避免一上来就乱建目录、乱写代码、乱开功能。

## 二、适用范围

仅适用于 V1 第一阶段。

当前阶段目标固定为“出纳资金日报最小可用闭环”。

## 三、Claude Code 每次开工前必须先读的文件顺序

1. `docs/00_project_constitution.md`
2. `docs/01_v1_scope_and_order.md`
3. `docs/02_user_constraints.md`
4. `docs/03_tech_constraints.md`
5. 当前轮次要执行的模块文档
6. 当前轮次涉及的契约文档
7. 当前轮次涉及的测试文档

若文件之间冲突，以范围文档、技术边界、契约文档为准。

## 四、正式开工顺序

### 第 1 轮
只做项目骨架与可运行环境。

输出目标：
- 前后端目录建立完成
- FastAPI 可启动
- Vue 页面可打开
- 前后端基础联通
- SQLite 初始化脚手架存在

禁止：
- 禁止开始做具体业务逻辑
- 禁止先写花哨页面
- 禁止先引入未来模块

### 第 2 轮
只做数据库与主数据中心。

输出目标：
- 核心表创建
- 主数据 CRUD 打通
- 账户初始化可用
- 期初余额与余额日期可保存

### 第 3 轮
做 AI 配置与 Agent 骨架。

输出目标：
- AI 配置页可用
- Provider 配置存储可用
- shared / master / parser-assistant 目录可自动创建

### 第 4 轮
做银行流水导入与模板识别。

输出目标：
- 文件上传可用
- 已有模板命中可用
- 新模板识别骨架可用
- 预览写入链路可用

### 第 5 轮
做手工流水双轨。

输出目标：
- 系统内快速录入可用
- 多主体 Excel 上传可用
- 上传结果预览与手动维护可用

### 第 6 轮
做基础数据表与日报生成。

输出目标：
- 标准资金事件入库
- 区间日报生成
- 基础数据表可核对
- 多日补做与重算可用

### 第 7 轮
做首页总控台、导出、打印、看板、备份、回滚。

输出目标：
- 首页工作总览可用
- 导出 Excel 可用
- 打印样式可用
- 批次回滚可用
- 基础看板可用

### 第 8 轮
只做测试修正。

输出目标：
- 模块测试全部跑通
- E2E 验收通过
- 真实样本回归通过

## 五、Claude Code 每轮输出格式

每轮必须输出：
- 本轮读了哪些文档
- 本轮改了哪些代码文件
- 本轮新增了哪些数据结构或接口
- 本轮还有哪些风险
- 本轮是否达到进入下一轮条件

## 六、禁止行为

- 不得自行扩大 V1 范围
- 不得擅自更换技术栈
- 不得先写页面壳后补业务逻辑
- 不得跳过契约文档直接写接口
- 不得在没有测试的情况下宣称完成

## 七、完成判定

只有当 V1 所有模块执行文档、契约文档、测试文档对应的完成标准都满足后，才允许进入“V1 已完成”状态。
