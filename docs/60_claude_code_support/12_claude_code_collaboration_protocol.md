# Claude Code 协作协议

## 一、作用

这份文档不是给产品看的，是给 Claude Code 和你之间协作用的。

它解决的是：每一轮开发交流时，Claude Code 应该怎么汇报、怎么停顿、什么时候能继续、什么时候必须先等你确认。

## 二、每轮任务输入格式

你给 Claude Code 的任务，必须至少包含：
- 当前模块名
- 当前执行文档名
- 当前契约文档名
- 当前测试目标
- 本轮明确不做什么

## 三、Claude Code 每轮必须输出什么

### 1. 阅读确认
必须先说本轮阅读了哪些文档。

### 2. 实施计划
必须说：
- 本轮准备改哪些文件
- 本轮准备新增哪些文件
- 本轮准备实现哪些接口 / 页面 / 表结构

### 3. 完成反馈
必须说：
- 实际改了哪些文件
- 遇到哪些风险
- 哪些功能已达到测试条件
- 哪些地方还未达标

### 4. 禁止输出
- 禁止只说“已完成”
- 禁止不列文件清单
- 禁止不列风险
- 禁止不列下一步条件

## 四、什么时候可以继续下一轮

只有同时满足下面条件才能继续：
- 当前模块的完成标准满足
- 对应测试场景至少跑过一次
- 没有关键阻塞问题悬空

## 五、什么时候必须停下

出现下面情况必须停下：
- 文档冲突
- 技术边界冲突
- 样本与期望输出冲突
- 当前模块未达成却想跳下一模块
- 发现需求中存在关键歧义

## 六、你每轮最好补充给 Claude Code 的材料

- 当前页面截图
- 当前样本文件
- 当前预期导出结果
- 当前认为最容易跑偏的点

## 七、目标

让 Claude Code 变成严格执行者，而不是自由发挥型写手。
