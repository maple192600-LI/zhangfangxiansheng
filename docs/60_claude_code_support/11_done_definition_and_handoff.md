# V1 完成定义与交付清单

## 一、作用

这份文档用于防止 Claude Code 把“代码写了”当成“V1 完成”。

V1 是否完成，必须按统一交付口径判断。

## 二、V1 必须同时交付的内容

### 1. 代码交付
- backend 可运行
- frontend 可运行
- 本地一键启动脚本可运行
- 数据库初始化脚本可运行

### 2. 文档交付
- 生效需求文档
- 技术边界文档
- 模块执行文档
- 契约文档
- 测试文档
- 本手册

### 3. 页面交付
- 首页总控台
- 主数据中心
- 网银导入页
- 手工流水页
- 上传结果预览页
- 基础数据表页
- 日报查看与导出页
- 备份恢复页

### 4. 数据与规则交付
- SQLite 数据库文件初始化成功
- 至少 1 套 seed 数据
- 至少 2 份银行样本可跑通
- 至少 1 份多主体手工总表样本可跑通
- 手工字段池初始化数据存在

### 5. 测试交付
- 模块测试全部通过
- E2E 验收通过
- 真实样本回归通过

## 三、V1 不算完成的情况

出现下面任意一种，都不算完成：
- 页面壳在，接口没接
- 导入能做，日报算不对
- 手工流水能录，基础数据表不稳定
- 导出能点，但导出结果错乱
- 没有回滚和备份
- 没有真实样本测试
- 没有完成文档留存

## 四、最终交接包应包含

```text
release_v1/
├── app/
├── docs/
├── samples/
├── exports_example/
├── backup_example/
└── release_note.md
```

## 五、release_note 必须写清

- 当前版本号
- 本版本完成范围
- 未完成范围
- 已知风险
- 启动方式
- 备份恢复方式
- 样本测试通过情况
