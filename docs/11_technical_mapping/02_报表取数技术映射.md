# 报表取数技术映射

> 把每张报表的计算逻辑映射到代码函数和 SQL 查询。

---

## 通用查询模式

所有报表查询共同的基础：

- **主表**：`fund_events`
- **关联表**：`accounts`（通过 `account_code`），`entities`（通过 `entity_code`）
- **状态过滤**：`fund_events.state = '正常'`
- **日期过滤**：`fund_events.business_date BETWEEN :start AND :end`
- **期初余额**：`base_data_service.get_opening_balance(db, account_id, start_date)`

---

## 各报表技术映射

### 资金日报

- **Service**：`report_service.daily_report(db, start_date, end_date, entity_id)`
- **文件**：`backend/services/report_service.py:19-90`
- **逻辑**：
  1. 查 `accounts` 表 `include_in_daily_report=True`
  2. 按 `account_code` 分组 SUM `amount_in` / `amount_out`
  3. 每个账户调 `get_opening_balance`
  4. 按 `entity_id` 汇总

### 现金日记账

- **Service**：`report_service.cash_journal(db, start_date, end_date, account_id)`
- **文件**：`backend/services/report_service.py:95-165`
- **逻辑**：
  1. 查 `accounts` 表
  2. 每个账户：调 `get_opening_balance` 获取期初
  3. 按 `business_date` 分组 SUM
  4. 逐日计算 `current_balance` 滚动余额
  5. 返回 `blocks` 数组（每个账户一个 block）

### 账户余额表

- **Service**：`report_service.account_balance(db, start_date, end_date, entity_id)`
- **文件**：`backend/services/report_service.py:170-264`
- **逻辑**：同日报，但不按法人汇总，逐账户展示 + 法人小计行 (`is_subtotal=True`)

### 收入/支出明细

- **Service**：`report_service.income_list` / `expense_list`
- **文件**：`backend/services/report_service.py:269-349`
- **逻辑**：`_direction_list` 通用函数，`amount_in > 0` 或 `amount_out > 0` 筛选 + 分页

### 资金余额表

- **Service**：`report_service.major_balance`
- **文件**：`backend/services/report_service.py:354-447`
- **区别**：只筛选 `instrument_type='银行存款'` 的账户

### 月度对账表

- **Service**：`report_service.month_check(db, year, month, entity_id)`
- **实现**：自动计算年月的起止日期，调用 `account_balance`

### 周报/月报/年报

- **Service**：`report_service.week_report` / `month_report` / `year_report`
- **实现**：日期范围转换后调用 `daily_report`

## 导出技术链路

1. 前端调用 `export.js` → `POST /api/export` → `backend/api/export.py`
2. `export_service.generate_export` 根据参数调用对应的 report_service 函数获取数据
3. 用 `openpyxl` 生成 Excel，返回 Blob
4. 前端用 `URL.createObjectURL` + `<a>` 下载
5. 文件名由 `frontend/src/utils/reportFilename.js` 的 `getReportFilename` 统一生成

---

## 前端模板列渲染

1. `useTemplateColumns(reportType)` composable 从 `/api/report-templates?report_type=xxx` 获取模板
2. 有模板 → 使用模板的 `columns_json` 作为列配置
3. 无模板 → 使用各页面内置的 `DEFAULT_COLUMNS`
4. 有 `layout_json` → CashJournal 使用完整的 Excel 布局渲染
5. 有 `source_file_path` → 渲染原始 Excel HTML（`excel-host`）

## 期初余额计算

**函数**：`base_data_service.get_opening_balance(db, account_id, start_date)`

**逻辑**：
1. 获取账户的 `initial_balance` 和 `balance_date`
2. 查询 `balance_date` 到 `start_date - 1天` 期间该账户的所有正常流水的 `SUM(amount_in) - SUM(amount_out)`
3. 期初 = `initial_balance + 净变动`

**重建余额**：`base_data_service.rebuild_rolling_balance` 重新计算所有账户的 `rolling_balance` 字段。
