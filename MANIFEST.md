# MANIFEST
下面列出总包中的主要文件与用途。

## 根目录
- `README.md`：总包说明、阅读顺序、使用建议
- `GET_STARTED.md`：最快开工路径

## docs/00_governance
- 上位约束层，控制项目边界、当前阶段范围、技术栈、用户边界、Claude Code 执行边界

## docs/10_product_design
- 产品与设计沉淀层，统一项目理解、首页信息架构、资金板块结构、手工流水机制、字段方案

## docs/20_execution
- 可执行模块文档，Claude Code 直接按此开工

## docs/30_contracts
- 契约层，统一数据库、字段、API、页面状态与异常处理

## docs/40_expected_outputs
- 期望输出层，告诉开发和测试“什么结果算对”

## docs/50_tests
- 模块测试、端到端测试、真实样本回归测试

## docs/60_claude_code_support
- Claude Code 的开工手册、目录规范、样本计划、交付定义、协作协议

## templates
- 手工多主体总表模板、模板规范、前端确认版原型

## samples
- 样本目录骨架，等你后续补真实脱敏样本

## references
- 原始需求、风格规范、交互边界和截图参考，不直接驱动开发，但能帮助回溯设计来源
