## ARCHITECTURE

每个业务域按同一结构实现：

```text
DomainView
-> domain api module
-> backend api router
-> domain service
-> dynamic_objects / domain tables
-> audit_logs
```

业务页面只处理业务动作，不管理 Agent 配置和 Skill 草稿。

业务页面的用户可见文本必须中文优先。页面内 Agent 入口可以接收中文命令，但必须通过 `command_id` 路由到底层动作。

## PAGE GROUPS

```text
资金：账户、流水、余额、日报入口
OCR：文件上传、识别任务、结果校对入口
贷款：贷款台账、还款计划、提醒入口
预算：预算表、资金计划、汇总入口
文件仓库：文件列表、留存策略、空间占用
规则与自动化：解析规则、字段映射、工作流绑定
连接器：CLI、API、MCP、ERP-OA 配置入口
系统管理：模型配置、权限、备份、日志
```

## IMPLEMENTATION METHOD

每个业务域必须单独任务单和 PR。

最小闭环要求：

```text
页面可打开
数据来自后端
至少一个真实用户动作
动作有 API 契约
错误中文可读
刷新后状态保留
页面可召唤 Agent
页面 Agent 命令返回中文过程说明
浏览器验收记录落文件
```

## CORE INTERFACES

每个业务域 service 至少提供：

```python
def list_domain_records(domain: str, filters: dict) -> DomainListResult:
    ...

def get_domain_summary(domain: str) -> DomainSummary:
    ...

def run_domain_action(domain: str, action: str, payload: dict) -> DomainActionResult:
    ...

def build_page_context(domain: str, selection: dict) -> PageAgentContext:
    ...
```

前端 API 至少提供：

```ts
export function listDomainRecords(domain: string, filters: Record<string, unknown>): Promise<DomainListResult>
export function getDomainSummary(domain: string): Promise<DomainSummary>
export function runDomainAction(domain: string, action: string, payload: unknown): Promise<DomainActionResult>
export function buildPageAgentContext(domain: string, selection: unknown): PageAgentContext
```
