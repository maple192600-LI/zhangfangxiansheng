# 08_项目护栏检查规范

## READ WHEN

准备完成任务、创建 PR、合并、打包、修复失败任务后必须读取。

## GUARD SCRIPT

开发阶段必须提供：

```text
dev/scripts/check_project_guard
```

该脚本用于检查 AI 是否破坏项目边界。没有护栏检查结果，不得声明任务完成。

脚本实现规格见：

```text
14_项目护栏检查脚本规格.md
```

## REQUIRED CHECKS

护栏检查至少覆盖：

```text
文件地图：
  不存在未登记顶层目录
  app/ 内没有 PRD、开发规范、测试样本、运行数据库、临时文件

临时产物：
  data/tmp/ 可清理
  data/downloads/ 可清理
  临时导出、截图、日志未进入提交

环境隔离：
  .venv/ 未提交
  node_modules/ 未提交
  data/ 未提交

替换实现清理：
  不存在并行替代命名绕过
  替换任务的原路由、原页面、原 API、原服务引用为 0

GitHub：
  当前实现任务不在 main
  失败分支未合并
  PR 中有浏览器验收记录
```

## MUST

- 完成说明必须写出护栏检查结果。
- 护栏检查失败时，先修复失败项，不要继续叠加功能。
- 如果护栏脚本尚未实现，首次进入代码开发阶段必须先实现最小版本。
- 用户可见功能仍必须做浏览器验收；护栏检查不能替代浏览器验收。

## NEVER

- 不要因为护栏检查失败而删除检查项。
- 不要把失败项写成“已知问题”后继续合并。
- 不要手工口头声明清理完成而不运行检查。

## COMPLETION LINE

完成声明必须包含：

```text
项目护栏检查：通过 / 失败 / 尚未实现，不得声明完成
```
