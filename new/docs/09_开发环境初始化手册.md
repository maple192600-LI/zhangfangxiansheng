# 09_开发环境初始化手册

## READ WHEN

创建开发仓库、初始化本地开发目录、配置 GitHub 私有仓库、第一次让 AI Coding 工具开工前必须读取。

## GOAL

使用本文件体系作为开发依据，初始化仓库和本地开发环境。

## DEFAULT MODEL

```text
GitHub Private Repo：代码和文档保险箱
Local Workspace：实际开发、运行、浏览器验收
Task Branch：每个任务的临时开发线
PR：每个任务的验收单
```

第二设备开发是可选场景；默认只要求一台本地电脑完成开发和验收。

## INITIAL REPOSITORY

目标仓库初始目录：

```text
AGENTS.md
app/
docs/
dev/
data/
skills/
workspaces/
.gitignore
README.md
```

## COPY DOCUMENTS

推荐放置：

```text
AGENTS.md                                -> AGENTS.md
README.md                                -> README.md
CLAUDE.md                                -> CLAUDE.md
PRD_账房先生_本地财务Agent工作台.md        -> PRD_账房先生_本地财务Agent工作台.md
docs/                                    -> docs/
rules/                                   -> rules/
```

复制后，根目录 `AGENTS.md` 必须能直接路由到 `docs/` 与 `rules/`。

同时从 Anthropic skills 仓库或当前 AI Coding 工具内置来源准备 `skill-creator`，作为全局 Skill 放到：

```text
skills/skill-creator/
```

`skill-creator` 必须保持 `SKILL.md` 兼容结构，并作为全局 Skill 识别。不得把任何外部工具的 Skill 目录固定为产品主 Skill 目录。

## GITIGNORE BASELINE

`.gitignore` 必须至少包含：

```text
.venv/
node_modules/
data/**
!data/.gitkeep
*.log
*.tmp
*.cache
*.sqlite
*.db
test-results/
playwright-report/
dist/
build/
coverage/
```

用户主动放入 `dev/fixtures/` 的测试样本可以提交。

## FIRST BRANCH FLOW

```text
1. 创建 GitHub Private 仓库
2. 克隆到本地
3. 建立初始目录
4. 复制已确认的文档
5. 创建 .gitignore
6. 提交 initial docs
7. 从 main 创建第一个任务分支
8. 先实现项目护栏检查最小版本
9. 再开始产品骨架开发
```

## FIRST CODE TASK

第一项代码任务必须是：

```text
dev/scripts/check_project_guard
```

最小版本必须检查：

- 未登记顶层目录。
- `app/` 中是否混入文档、测试样本、运行数据、临时文件。
- `data/`、`.venv/`、`node_modules/` 是否会进入提交。
- 是否存在并行替代命名。
- 当前实现任务是否直接在 `main`。

## NEVER

- 不要把未确认文件复制进目标仓库。
- 不要复制未确认测试文件。
- 不要复制未确认运行数据。
- 不要在 `main` 直接开发产品功能。
- 不要在护栏检查实现前声明代码开发流程可用。
- 不要跳过 `skills/skill-creator/` 的预置和登记。

## DONE

初始化完成必须满足：

- GitHub 仓库是 Private。
- 本地仓库可以正常提交和推送。
- 目录符合文件地图。
- 根 `AGENTS.md` 指向正式文档位置。
- `skills/skill-creator/SKILL.md` 已存在并登记来源、版本或 hash。
- `.gitignore` 阻止运行产物和依赖目录进入提交。
- 项目护栏检查最小版本已存在。
- 第一个 PR 只包含初始化内容，不混入产品功能。
