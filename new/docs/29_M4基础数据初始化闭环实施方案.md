## ARCHITECTURE

M4 采用导入批次架构：

```text
DataImportView
-> api/imports.ts
-> api/imports.py
-> services/import_service.py
-> core/files/store.py
-> core/excel/preview.py
-> import_batches / import_stage_rows / dynamic_object_rows
```

文件仓库只保存原文件和元数据，正式业务数据只在用户确认后进入 `dynamic_object_rows`。

## FLOW

```text
1. 用户选择业务对象和 Excel 文件
2. 上传接口计算 sha256 并保存文件元数据
3. 创建 import_batch，状态 uploaded
4. preview 读取前 50 行并写 preview_rows_json
5. 用户选择表头行和字段映射
6. mapping 保存 Excel 原列名到 field_key 的映射
7. stage 按 field_key 校验必填和类型
8. 校验失败写 error_json
9. 校验成功写 import_stage_rows
10. 用户确认后写 dynamic_object_rows
11. 写 audit_logs
12. 前端刷新对象行列表
```

## FILE STORAGE

上传文件保存到：

```text
data/files/
```

保存规则：

```text
按 sha256 去重
数据库保存元数据
文件内容不进 SQLite
文件不提交 GitHub
预览临时产物放 data/tmp/
```

文件 hash 计算必须基于文件内容，不基于文件名。

## API

### POST /api/imports/upload

上传 Excel 文件并创建导入批次。

请求：

```text
multipart/form-data
file
object_id
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "batch_id": "batch_xxx",
    "file_id": "file_xxx",
    "status": "uploaded"
  }
}
```

### POST /api/imports/{batch_id}/preview

读取 Excel 前 50 行，返回 sheet、表头候选、预览行。

不得把文件名、Sheet 名、固定列号写成判断逻辑。

### POST /api/imports/{batch_id}/mapping

保存字段映射。

请求：

```json
{
  "header_row_index": 1,
  "mapping": {
    "Excel列名": "field_key"
  }
}
```

### POST /api/imports/{batch_id}/stage

根据映射写入暂存行。

必须校验：

```text
必填字段
字段类型
field_key 是否属于 object_id
```

### POST /api/imports/{batch_id}/confirm

用户确认后从暂存行写入 `dynamic_object_rows`。

确认时必须写审计日志。

### GET /api/objects/{object_id}/rows

查询动态对象数据行。

### PATCH /api/object-rows/{row_id}

修改动态对象数据行。

必须写审计日志。

## FIELD MAPPING RULE

字段映射必须保存：

```text
Excel 原列名
field_key
field_id
data_type
required
```

不得只保存展示名。

## ROW VALUE RULE

`values_json` 使用 field_key 作为键：

```json
{
  "project_name": "项目A",
  "project_code": "P001"
}
```

不得使用 Excel 原列名或字段展示名作为正式键。
