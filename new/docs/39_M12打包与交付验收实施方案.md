# 39_M12打包与交付验收实施方案

## READ WHEN

执行 `24_产品代码开发路线图.md` 的 M12 打包与交付验收时读取。

## GOAL

形成用户可运行的本地应用交付包，并确保交付包不包含开发文档、测试样本、运行数据库、临时产物和依赖缓存。

## MUST READ

```text
docs/01_项目文件地图与交付边界.md
docs/02_测试与验收规范.md
docs/05_开发产物与清理规范.md
docs/08_项目护栏检查规范.md
docs/15_浏览器验收记录规范.md
docs/25_技术栈与工程架构蓝图.md
rules/07_文件边界与交付白名单.md
rules/08_临时文件与开发产物清理.md
rules/12_项目护栏检查.md
```

## REFERENCE BASIS

| 参考对象 | 借鉴点 | 本方案落点 |
|---|---|---|
| FastAPI 静态挂载 | 前端构建后由后端服务托管 | `app/packaging` |
| PyInstaller | 本地可执行交付 | 桌面交付阶段 |
| 常见 release 白名单 | 只打包运行必需文件 | `release_manifest.json` |

## ARCHITECTURE

```text
npm run build
-> frontend dist
-> backend 静态挂载
-> package script
-> release_manifest
-> clean check
-> browser full regression
```

## CREATE FILES

```text
app/packaging/build_frontend.ps1
app/packaging/build_backend.ps1
app/packaging/package_local.ps1
app/packaging/release_manifest.json
app/packaging/verify_release.py
dev/reports/release_acceptance/.gitkeep
```

## RELEASE ALLOWLIST

交付包只允许包含：

```text
后端运行文件
前端构建产物
必要启动脚本
必要默认配置样板
空数据目录占位
许可证或说明文件
```

不得包含：

```text
docs/
rules/
dev/
tests/
测试样本
运行数据库
日志
截图
临时导出
.venv/
node_modules/
```

## CORE INTERFACES

```python
def build_release_manifest(root: Path) -> ReleaseManifest:
    ...

def verify_release_package(path: Path) -> ReleaseCheckResult:
    ...
```

## FLOW

```text
1. 运行后端测试
2. 运行前端 typecheck 和 build
3. 清理 data/tmp、日志、下载临时文件
4. 构建前端静态文件
5. 后端挂载静态文件
6. 生成 release_manifest
7. 打包本地交付目录
8. verify_release 检查白名单
9. 启动交付包
10. 浏览器完整回归
```

## TESTS

```text
后端测试通过
前端 build 通过
verify_release 阻断 docs/
verify_release 阻断 dev/
verify_release 阻断运行数据库
交付包可启动
```

## BROWSER ACCEPTANCE

```text
启动交付包
打开首页
检查健康状态
打开数据中心
打开 Agent 中心
上传样本或使用允许样本完成回归
导出文件
刷新页面
确认中文错误提示正常
```

## EXIT CONDITION

```text
交付包可启动：
交付包白名单通过：
不包含开发文档：
不包含测试样本：
不包含运行数据库：
完整浏览器回归通过：
项目护栏检查通过：
```

## DO NOT

```text
不要把 docs/ 打进用户交付包
不要把 dev/ 打进用户交付包
不要把测试样本打进交付包
不要把运行数据库打进交付包
不要跳过交付包启动验证
```
