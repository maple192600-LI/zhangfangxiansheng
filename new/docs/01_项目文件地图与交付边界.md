# 01_项目文件地图与交付边界

## READ WHEN

创建项目、移动文件、新增目录、写测试样本、打包交付、清理文件前必须读取。

## TARGET LAYOUT

正式仓库使用固定边界：

```text
app/
  backend/
  frontend/
  packaging/

.github/
  ISSUE_TEMPLATE/

docs/

dev/
  fixtures/
  scripts/
  reports/

data/
  databases/
  files/
  logs/
  tmp/
  downloads/

skills/
  skill-creator/
  index.json

workspaces/
  default/
    skills/
    files/
    rules/
    workflows/
    reports/
```

## DIRECTORY RULES

| 目录 | 内容 | 是否进入用户交付包 |
|---|---|---|
| `app/` | 应用源码、前端、后端、打包入口 | 是 |
| `.github/` | GitHub Issue、PR 模板和工作流配置 | 否 |
| `docs/` | PRD、架构、规范、验收、ADR | 否 |
| `dev/fixtures/` | 用户确认可用于测试的样本、黄金样本 | 否 |
| `dev/scripts/` | 检查、清理、打包、开发脚本 | 否 |
| `dev/reports/` | 可重新生成的测试报告摘要 | 否 |
| `data/` | 本机运行数据库、日志、缓存、临时文件 | 否 |
| `skills/` | 全局 Skill，所有工作区和 Agent 默认可见 | 是 |
| `workspaces/` | 工作区文件、规则、工作流、工作区 Skill | 是 |

## MUST

- 新增目录前先判断属于上表哪一类。
- 不确定位置时先停止并说明，不要自由创建目录。
- `app/` 内不得放 PRD、开发规范、测试样本、运行数据库、临时文件。
- `.github/` 只放 GitHub 模板和必要的仓库自动化配置。
- 全局 Skill 直接放 `skills/<skill-name>/`，不得增加中间层目录。
- 工作区 Skill 放 `workspaces/<workspace_id>/skills/<skill-name>/`。
- 用户主动放入 `dev/fixtures/` 的样本视为可用于开发和提交。
- 用户交付包只能从白名单生成，默认只包含 `app/` 构建结果和必要运行文件。
- 新增长期保留目录时，必须同时更新本文件。

## NEVER

- 不要在仓库根目录随手创建 `temp`、`test`、`output`、`random_files`、`backup`。
- 不要把浏览器下载文件、导出测试文件、调试截图放进 `app/`。
- 不要把运行时数据库提交到 GitHub。
- 不要把开发文档、测试样本、临时报告打进用户交付包。

## DELIVERY CHECK

交付前必须确认：

- `app/` 可独立构建。
- `data/` 内运行产物不进入 GitHub 提交，最多保留 `.gitkeep`。
- `dev/` 不进入用户交付包。
- 没有未登记的新顶层目录。
- 没有临时文件残留在产品目录。
