# 10_开发阶段删除语义规范

## READ WHEN

开发阶段用户要求删除功能、页面、按钮、接口、模板、测试文件、上传样本、规则草稿、工作流草稿时必须读取。

## DEFAULT MEANING

开发阶段的“删除”默认表示彻底删除，不是隐藏、停用、归档或只删前端入口。

除非用户明确说“先隐藏”“暂时保留”“停用”“归档”，否则必须按彻底删除处理。

## DELETE FEATURE

删除功能必须清理全链路：

```text
前端页面
前端路由
菜单入口
按钮入口
权限配置
前端 API 调用
后端 API 路由
后端 service
数据库表 / 字段 / migration
种子数据
测试用例
测试样本
文档引用
工作流节点
Skill / 规则引用
导出 / 报表引用
```

完成前必须搜索功能名、路由名、接口路径、组件名、service 名、表名、字段名，残留必须为 0；允许保留的残留必须逐条说明原因。

## DELETE DEV FILE OR TEMPLATE

开发阶段删除上传文件、模板、样本、规则草稿、工作流草稿时，默认删除：

```text
文件本体
文件元数据
解析结果
预览缓存
关联规则草稿
关联工作流草稿
测试输出
前端列表记录
后端数据库记录
```

如果该文件已被写入测试样本目录，删除前必须确认是否是用户要求保留的样本。

## NEVER

- 不要只删除前端入口。
- 不要只隐藏菜单。
- 不要只删除列表记录而保留后端数据。
- 不要只删除文件本体而保留元数据。
- 不要只删除数据库记录而保留文件仓库内容。
- 不要把被删除功能留给后续任务继续引用。
- 不要在后续开发中从残留代码、残留文档、残留测试里恢复已删除内容。

## REQUIRED OUTPUT

删除任务完成时必须输出：

```text
删除对象：
已删除前端：
已删除后端：
已删除数据库 / migration：
已删除测试：
已删除样本 / 文件：
已删除文档引用：
残留搜索关键词：
残留结果：0 / 有残留并逐条说明
浏览器验收：
```

## BROWSER CHECK

用户可见删除必须验证：

- 页面入口消失。
- 直接访问原路由不可用或跳转到合理页面。
- 原按钮不可见。
- 原列表项不可见。
- 原接口不再被前端调用。
- 刷新页面后仍不存在。
