## TASK 1：仓库初始化与首次提交

### 用户原始要求

使用 GitHub Private 仓库管理项目开发；AI Coding 工具负责创建目录、复制已确认文档、提交和创建 PR；用户通过 GitHub 页面和本地浏览器验收结果确认。

### 本次目标

建立可开发、可提交、可同步、可继续验收的仓库基础。

### 不做范围

- 不实现产品功能。
- 不安装业务依赖。
- 不创建运行数据库。
- 不创建业务 Skill。
- 不创建业务页面。

### 需要读取的规则卡

### 用户确认点

### 操作步骤

1. 确认 GitHub 登录和授权。
2. 创建 Private 仓库。
3. 在用户确认的位置克隆仓库。
4. 创建目标目录。
5. 复制 `AGENTS.md`、`README.md`、`CLAUDE.md`、PRD、`docs/`、`rules/`。
6. 创建 `.gitignore`。
7. 预置 `skills/skill-creator/SKILL.md`。
8. 创建 `skills/index.json`。
9. 创建 `workspaces/default/skills/index.json`。
10. 执行 `git status --short`。
11. 提交 `initial project foundation`。
12. 推送到 GitHub。
13. 创建首次 PR。

### 验收计划

- 后端自检：不涉及。
- 浏览器验收：打开 GitHub 仓库页面，确认仓库为 Private，确认目录存在。
- 项目护栏检查：本任务只确认 `.gitignore` 和目录边界；护栏脚本在 TASK 2 实现。
- 用户检查路径：
  1. 打开 GitHub 仓库首页。
  2. 查看仓库权限标识为 Private。
  3. 查看根目录文件。
  4. 查看 `docs/`、`rules/`、`skills/skill-creator/`。
  5. 查看首次 PR。

## TASK 2：实现项目护栏检查脚本

### 用户原始要求

开发过程必须防止文件乱放、依赖污染、临时产物残留、并行替代实现、删除残留、直接在主分支开发等问题；AI Coding 工具完成任务前必须运行护栏检查。

### 本次目标

实现可在本地运行的项目护栏脚本，作为后续任务完成前的强制检查。

### 不做范围

- 不实现产品功能。
- 不接入前端页面。
- 不安装业务依赖。
- 不创建运行数据库。
- 不创建测试样本。

### 必读文档

- `AGENTS.md`
- `docs/01_项目文件地图与交付边界.md`
- `docs/03_AI_Coding_开发协作规范.md`
- `docs/08_项目护栏检查规范.md`
- `docs/12_任务单与PR模板规范.md`
- `docs/14_项目护栏检查脚本规格.md`

### 需要读取的规则卡

- `rules/06_PR与完成定义.md`
- `rules/07_文件边界与交付白名单.md`
- `rules/08_临时文件与开发产物清理.md`
- `rules/09_问题修复与替换实现清理.md`
- `rules/11_GitHub分支与PR开发.md`
- `rules/12_项目护栏检查.md`
- `rules/15_任务单与PR模板.md`
- `rules/17_项目护栏脚本实现.md`

### 修改对象

- 前端：无。
- 后端：无。
- 数据库：无。
- 文档：必要时补充护栏检查说明。
- 测试：增加护栏脚本最小验证。
- 样本 / 文件：不新增业务样本。

### 必须创建

```text
dev/scripts/check_project_guard.py
dev/scripts/check_project_guard.ps1
dev/reports/project_guard_sample_output.md
```

`check_project_guard.ps1` 只负责调用 Python 脚本。

### 检查项

必须覆盖：

- 未登记顶层目录。
- `app/` 中混入文档、测试样本、运行数据、临时文件。
- `docs/` 中混入运行数据库、依赖目录。
- `dev/` 中混入产品运行数据库。
- `data/` 中存在待提交运行产物。
- `.gitignore` 缺少必需排除项。
- 根目录临时文件、调试截图、临时数据库、下载文件。
- 可疑并行替代实现命名。
- 删除任务缺少残留搜索关键词。
- 当前分支直接在 `main` 或 `master` 做实现任务。
- 根 `AGENTS.md` 缺失。
- `docs/`、`rules/` 缺失。
- `skills/skill-creator/SKILL.md` 缺失。
- 出现多余 Skill 目录层级。

### 输出格式

必须输出中文摘要和 JSON：

```text
项目护栏检查：通过 / 失败

检查项：
- 文件边界：通过 / 失败
- 提交排除：通过 / 失败
- 临时产物：通过 / 失败
- 并行替代实现：通过 / 失败
- 删除残留：通过 / 失败 / 不涉及
- Git 分支：通过 / 失败
- 文档入口：通过 / 失败
- Skill 目录：通过 / 失败

JSON:
{
  "ok": true,
  "checks": []
}
```

### 命令

```text
python dev/scripts/check_project_guard.py
```

Windows 可用：

```text
powershell -ExecutionPolicy Bypass -File dev/scripts/check_project_guard.ps1
```

### 验收计划

- 后端自检：运行护栏脚本，成功时 exit code 为 `0`。
- 失败验证：临时创建一个违规文件，运行脚本必须 exit code 为 `1`，然后删除该违规文件。
- 浏览器验收：不涉及用户页面。
- 项目护栏检查：运行 `python dev/scripts/check_project_guard.py`。
- 用户检查路径：
  1. 查看 `dev/scripts/check_project_guard.py`。
  2. 查看 `dev/scripts/check_project_guard.ps1`。
  3. 查看 `dev/reports/project_guard_sample_output.md`。
  4. 查看 PR 验证结果。
