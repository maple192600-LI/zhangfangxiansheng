## ARCHITECTURE

```text
External Client
-> CLI / Local API / MCP Adapter
-> ConnectorGateway
-> PermissionGuard
-> Service Layer
-> AuditLog
```

MCP 不作为唯一入口。CLI 和本地 API 必须先可用。

CLI 命令可以提供英文稳定命令和中文别名。中文别名解析后必须进入同一个 command_id。

## CLI COMMANDS

最小 CLI：

```text
zf status
zf objects list
zf workflows run --recipe <id> --input <file>
zf agents send --agent <id> --message <text>
```

中文别名：

```text
zf 状态
zf 对象 列表
zf 工作流 运行 --配方 <id> --输入 <file>
zf 智能体 发送 --智能体 <id> --消息 <text>
```

所有写入命令必须走权限和审计。

中文 CLI 只作为输入别名，内部仍解析为：

```text
status
objects.list
workflows.run
agents.send
```

## MCP STATE

MCP server 必须有状态：

```text
disconnected
connecting
connected
auth_required
error
```

只有 connected 状态可以列出工具和调用工具。MCP 工具必须进入 ConnectorGateway 和 PermissionGuard，不得直接暴露给 Agent。

## FLOW

```text
1. 用户创建连接器
2. 配置可读写范围
3. 测试连接
4. 外部系统发起读取或任务触发
5. ConnectorGateway 接收
6. PermissionGuard 判断
7. service 执行
8. audit_logs 记录
9. 返回结构化结果
```

## TESTS

```text
CLI 可读取状态
CLI 未授权写入被拒绝
API 连接器权限生效
MCP 适配层只暴露允许工具
外部写入写审计日志
连接失败返回中文错误
MCP 未连接时调用失败并显示中文原因
中文 CLI 别名和英文命令执行同一 command_id
```


## DO NOT

```text
不要把 MCP 作为唯一连接方式
不要让外部系统绕过 PermissionGuard
不要默认开放写入
不要把密钥写入日志
不要让 CLI 直接改数据库
```
