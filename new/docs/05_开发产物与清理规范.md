# 05_开发产物与清理规范

## READ WHEN

生成测试文件、导出文件、截图、日志、调试脚本、临时数据库、浏览器下载内容前必须读取。

## FILE CLASSES

| 类型 | 放置位置 | 任务结束 |
|---|---|---|
| 用户确认的测试样本 | `dev/fixtures/` | 保留 |
| 黄金样本期望输出 | `dev/fixtures/expected/` | 保留 |
| 开发脚本 | `dev/scripts/` | 保留 |
| 测试报告摘要 | `dev/reports/` | 可保留摘要 |
| 浏览器下载 / 导出测试文件 | `data/downloads/` | 删除 |
| 临时预览 / 中间文件 | `data/tmp/` | 删除 |
| 调试数据库 | `data/databases/dev/` | 删除或重建 |
| 运行日志 | `data/logs/` | 不提交 |

## MUST

- 任务开始前说明会产生哪些临时产物。
- 任务结束前删除可重建的临时文件。
- 需要长期保留的样本必须放入 `dev/fixtures/`。
- 浏览器验收记录写在完成说明或 PR 文本中，不靠一堆截图文件长期占位。
- 清理动作不得删除用户主动提供的测试样本。

## NEVER

- 不要把临时导出文件留在仓库根目录。
- 不要把测试下载文件提交到 GitHub。
- 不要把失败试跑文件留在产品目录。
- 不要为了“留证据”保留大量不可读临时文件。
- 不要删除 `dev/fixtures/` 中的用户样本，除非用户明确要求。

## COMPLETION CHECK

完成声明必须包含：

```text
临时文件：已清理 / 无临时文件 / 未清理并说明原因
测试样本：新增路径 / 未新增
运行产物：未进入 GitHub 提交
```
