# 04_工程环境与依赖隔离规范

## READ WHEN

安装依赖、创建数据库、启动服务、打包、配置开发环境、多设备继续开发前必须读取。

## MUST

- Python 依赖安装到项目内 `.venv`。
- Node 依赖安装到 `app/frontend/node_modules`。
- 数据库、日志、缓存、下载文件写入 `data/`。
- 开发脚本放入 `dev/scripts/`。
- 依赖版本写入锁定文件或清单文件。
- 如果启用第二设备开发，必须使用同一仓库、同一文档、同一依赖清单。
- 任何全局安装、系统环境变量修改、系统数据库服务安装，必须先得到用户明确同意。

## NEVER

- 不要安装到系统 Python。
- 不要把依赖装到用户主目录当作项目依赖。
- 不要要求用户手工改系统环境才能运行产品。
- 不要让开发数据库和用户真实运行数据库混在一起。
- 不要把 `node_modules`、`.venv`、缓存、日志提交到 GitHub。

## LOCAL PATHS

```text
.venv/
app/frontend/node_modules/
data/databases/
data/logs/
data/tmp/
data/downloads/
```

以上路径默认加入 `.gitignore`。

## CHECK

任务完成前必须确认：

- 依赖在项目隔离环境内。
- 本机全局环境未被修改，或已获得用户明确同意。
- 运行数据库在 `data/`。
- 临时文件在 `data/tmp/` 或 `data/downloads/`。
- GitHub 提交不包含本机环境产物。
