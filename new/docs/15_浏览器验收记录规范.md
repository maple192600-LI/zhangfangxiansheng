# 15_浏览器验收记录规范

## READ WHEN

实现用户可见功能、修复页面问题、上传文件、导出文件、删除入口、提交 PR 前必须读取。

## PURPOSE

浏览器验收必须模拟用户真实操作，并留下用户能看懂的记录。

后端测试、接口测试、单元测试不能替代浏览器验收。

## REQUIRED TOOL

AI Coding 工具必须使用真实浏览器操作工具。

```text
Claude Code: 必须使用 web-access skill
Codex: 必须使用 browser-use，或当前环境明确提供的等价浏览器操作 skill
Playwright: 只能作为补充自动化或记录工具，不能替代 web-access / browser-use 的真实操作验收
```

如果当前环境没有浏览器工具，必须写：

```text
未完成浏览器验收，不能声明用户可见功能完成。
```

## RECORD LOCATION

验收记录写入：

```text
dev/reports/browser_acceptance/
```

文件命名：

```text
YYYYMMDD_HHMM_<task-name>.md
```

截图、视频、下载文件默认放：

```text
data/downloads/
data/tmp/
```

任务结束后按临时产物规则清理。长期保留的证据只保留文字摘要或必要截图。

## MUST VERIFY

根据任务类型选择：

```text
页面能打开
按钮能点击
表单能提交
上传能完成
预览能显示
错误提示中文可读
数据刷新后仍存在
删除后刷新仍不存在
导出文件可打开或可解析
列表、筛选、详情、返回路径可用
相关页面联动正确
```

## FILE UPLOAD CHECK

涉及上传必须验证：

```text
选择文件
上传状态
识别 / 解析状态
预览结果
异常提示
确认或取消
结果在目标页面可查询
```

## EXPORT CHECK

涉及导出必须验证：

```text
点击导出
下载文件存在
文件可打开或可解析
关键字段和金额正确
文件名合理
导出记录可查
```

## DELETE CHECK

涉及删除必须验证：

```text
入口或列表项删除前可见
执行删除动作
确认弹窗或提示正确
删除后不可见
刷新后仍不可见
相关页面不再引用失效对象
```

## NEVER

- 不要只说“已测试”。
- 不要只跑接口或单元测试。
- 不要只打开首页。
- 不要用 Playwright 结果替代 Claude Code 的 web-access skill 操作记录。
- 不要跳过错误场景。
- 不要用假截图代替真实操作。
- 不要把下载文件、截图、视频长期留在产品目录。

## DONE

完成声明必须包含：

```text
浏览器验收记录：
结论：
用户复查路径：
未通过项：
```
