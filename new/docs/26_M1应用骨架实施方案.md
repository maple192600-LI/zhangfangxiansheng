## ARCHITECTURE

M1 只建立最小运行链路：

```text
浏览器
-> Vite dev server
-> Vue App
-> api/client.ts
-> FastAPI /api/health
-> 统一响应
```

后端只暴露健康检查，前端只验证真实接口连通。

## CORE INTERFACES

后端创建函数：

```python
def create_app() -> FastAPI:
    ...
```

统一成功响应：

```python
def ok(data: dict | None = None) -> dict:
    return {"code": 0, "message": "ok", "data": data or {}}
```

前端 API client：

```ts
export async function request<T>(path: string): Promise<T>
```

健康检查 API：

```ts
export async function getHealth(): Promise<HealthData>
```

## FLOW

```text
1. 后端 create_app 注册 health router
2. GET /api/health 返回统一响应
3. 前端启动时调用 getHealth
4. HomeView 根据真实响应显示后端状态
5. 请求失败时显示中文错误
6. 浏览器刷新后重新请求健康状态
```

## BACKEND CONTRACT

健康检查接口：

```text
GET /api/health
```

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "ok",
    "app": "账房先生",
    "mode": "development"
  }
}
```

失败响应走统一错误格式。

## BACKEND REQUIREMENTS

`app/backend/requirements.txt` 必须至少包含：

```text
fastapi
uvicorn[standard]
pydantic
pytest
httpx
```

安装后必须生成：

```text
app/backend/requirements.lock
```

## TESTS

必须有测试：

```text
app/backend/tests/test_health.py
```

测试目标：

```text
GET /api/health 返回 HTTP 200
响应 code = 0
响应 data.status = ok
```

运行命令：

```text
.venv/Scripts/python.exe -m pytest app/backend/tests
```

## START SCRIPTS

`dev/scripts/start_backend.ps1`：

```text
检查 .venv 是否存在
使用 .venv/Scripts/python.exe 启动 uvicorn
监听 127.0.0.1:8000
```

`dev/scripts/start_frontend.ps1`：

```text
进入 app/frontend
执行 npm run dev -- --host 127.0.0.1 --port 5173
```

脚本不得安装依赖；依赖安装必须由任务单单独说明。

## IMPLEMENTATION STEPS

1. 创建 M1 任务分支。
2. 读取 MUST READ 和 ROUTE RULES。
3. 创建后端目录和文件。
4. 创建统一响应函数。
5. 创建 `/api/health`。
6. 写后端健康检查测试。
7. 创建前端 Vite + Vue 结构。
8. 创建 API client。
9. 创建首页和主布局。
10. 首页调用 `/api/health`。
11. 创建启动脚本。
12. 安装依赖到隔离目录。
13. 生成依赖锁定文件。
14. 运行后端测试。
15. 运行前端 typecheck 和 build。
16. 启动后端和前端。
17. 使用浏览器打开 `http://127.0.0.1:5173/`。
18. 验证首页显示后端状态。
19. 写浏览器验收记录。
20. 运行项目护栏检查。
21. 提交 PR。
