# 14_项目护栏检查脚本规格

## READ WHEN

实现、修改、运行 `dev/scripts/check_project_guard` 前必须读取。

## PURPOSE

把开发规范变成机器检查。护栏脚本失败时，任务不得声明完成，不得创建可合并 PR。

## REFERENCE BASIS

| 参考对象 | 采用能力 | 本方案落点 |
|---|---|---|
| pre-commit 类检查 | 本地提交前快速失败 | `check_project_guard.py` |
| CI lint 输出 | 中文摘要 + JSON 机器结果 | 输出格式和 exit code |
| AI Coding 防漂移规则 | 文件边界、临时产物、替代实现、删除残留 | REQUIRED CHECKS |

不采用：

```text
不依赖远程 CI 才发现问题
不把检查结果写成只有人能看的长文本
不在护栏脚本中修改业务文件
```

## SCRIPT LOCATION

```text
dev/scripts/check_project_guard
```

如果目标开发环境需要平台兼容，可以提供：

```text
dev/scripts/check_project_guard.py
dev/scripts/check_project_guard.ps1
dev/scripts/check_project_guard.sh
```

至少必须有一个用户当前系统可运行的入口。

## COMMAND

推荐命令：

```text
python dev/scripts/check_project_guard.py
```

支持参数：

```text
python dev/scripts/check_project_guard.py
python dev/scripts/check_project_guard.py --task-type init --allow-main-init
python dev/scripts/check_project_guard.py --task-type feature
python dev/scripts/check_project_guard.py --task-type delete --delete-keywords "功能名,路由名,接口路径,组件名"
python dev/scripts/check_project_guard.py --json-only
```

参数规则：

| 参数 | 含义 | 默认 |
|---|---|---|
| `--task-type` | `init` / `feature` / `fix` / `delete` | `feature` |
| `--allow-main-init` | 只允许初始化任务在 `main` 或 `master` 执行 | `false` |
| `--delete-keywords` | 删除任务必须提供的残留搜索关键词，逗号分隔 | 空 |
| `--json-only` | 只输出 JSON，供 CI 或其他工具读取 | `false` |

## OUTPUT FORMAT

输出必须是中文摘要 + 机器可读 JSON。

```text
项目护栏检查：通过 / 失败

检查项：
- 文件边界：通过 / 失败
- 提交排除：通过 / 失败
- 临时产物：通过 / 失败
- 并行替代实现：通过 / 失败
- 删除残留：通过 / 失败 / 不涉及
- Git 分支：通过 / 失败
- 文档入口：通过 / 失败
- Skill 目录：通过 / 失败
- 模板一致性：通过 / 失败 / 不涉及

JSON:
{
  "ok": true,
  "checks": [
    {
      "id": "file_boundary",
      "name": "文件边界",
      "status": "pass",
      "severity": "error",
      "messages": [],
      "paths": []
    }
  ]
}
```

失败时必须列出具体路径和原因。

## RESULT SCHEMA

JSON 顶层结构：

```json
{
  "ok": true,
  "task_type": "feature",
  "branch": "codex/task-name",
  "checks": [],
  "summary": {
    "passed": 0,
    "failed": 0,
    "warning": 0,
    "skipped": 0
  }
}
```

`checks[]` 每项必须包含：

```json
{
  "id": "stable_snake_case_id",
  "name": "中文检查名",
  "status": "pass | fail | warning | skipped",
  "severity": "error | warning | info",
  "messages": ["中文原因"],
  "paths": ["具体路径"]
}
```

`severity = error` 且 `status = fail` 时，脚本 exit code 必须为 `1`。

## ALLOWED ROOT CONTENT

允许的根目录和根文件：

```text
AGENTS.md
README.md
CLAUDE.md
PRD_账房先生_本地财务Agent工作台.md
.gitignore
.github/
app/
docs/
rules/
dev/
data/
skills/
workspaces/
```

根目录出现其他文件或目录时，必须失败。

## REQUIRED GITIGNORE LINES

`.gitignore` 必须逐行包含：

```text
.venv/
node_modules/
data/**
!data/.gitkeep
*.log
*.tmp
*.cache
*.sqlite
*.db
test-results/
playwright-report/
dist/
build/
coverage/
```

缺少任意一行时，`gitignore_required` 检查失败。

## REQUIRED CHECKS

### 1. 文件边界

必须检查：

```text
只允许登记过的顶层目录
app/ 不包含 PRD、规范、测试样本、运行数据库、临时文件
docs/ 不包含运行数据库、依赖目录
dev/ 不包含产品运行数据库
data/ 不包含待提交运行产物
```

判定规则：

| 检查 | 失败条件 |
|---|---|
| `root_allowlist` | 根目录存在未登记目录或文件 |
| `app_boundary` | `app/` 内出现 `.md` 规范、测试样本、`.sqlite`、`.db`、`.log`、`.tmp`、截图、导出文件 |
| `github_boundary` | `.github/` 内出现运行数据、依赖目录、业务源码、测试样本、下载文件 |
| `docs_boundary` | `docs/` 内出现数据库、依赖目录、运行日志、浏览器报告 |
| `dev_boundary` | `dev/` 内出现产品运行数据库 |
| `data_boundary` | `data/` 内除 `.gitkeep` 外存在会被 Git 提交的文件 |

### 2. 提交排除

必须检查以下内容不会进入提交：

```text
.venv/
node_modules/
data/**
!data/.gitkeep
*.log
*.tmp
*.cache
*.sqlite
*.db
test-results/
playwright-report/
dist/
build/
coverage/
```

判定规则：

- `.gitignore` 缺少 REQUIRED GITIGNORE LINES，失败。
- 运行 `git status --short --ignored`。
- 如果 `.venv/`、`node_modules/`、`data/`、`*.sqlite`、`*.db`、`*.log` 以可提交状态出现，失败。
- 如果 `data/.gitkeep` 以外的 `data/` 文件可提交，失败。

### 3. 临时产物

必须检查：

```text
仓库根目录没有临时导出、调试截图、临时数据库、下载文件
app/ 内没有临时产物
data/tmp/ 和 data/downloads/ 只允许存在可清理文件，不允许进入提交
```

根目录和 `app/` 内禁止出现：

```text
*.tmp
*.cache
*.log
*.sqlite
*.db
*.png
*.jpg
*.jpeg
*.gif
*.webp
*.xlsx
*.xls
*.csv
*.zip
debug_*
tmp_*
temp_*
screenshot_*
download_*
```

例外：

- `dev/fixtures/` 中用户确认的测试样本允许存在。
- `dev/reports/` 中可复查报告允许存在。
- `data/.gitkeep` 允许存在。

### 4. 并行替代实现

必须检查可疑命名：

```text
*_copy*
*_bak*
*_backup*
*_temp*
*_tmp*
*_2
*_final
*_fix
*_fixed
legacy
deprecated
```

命中规则：

- 在 `app/` 内命中，默认失败。
- 在 `docs/` 或 `dev/reports/` 内命中，默认 warning。
- 失败项必须列出具体路径。

### 5. 删除残留

如果任务单声明涉及删除，必须读取任务单中的残留搜索关键词，并检查：

```text
功能名
路由名
接口路径
组件名
service 名
表名
字段名
菜单名
测试名
```

无任务单或无关键词时，删除任务不得通过护栏检查。

判定规则：

- `--task-type delete` 时，`--delete-keywords` 不能为空。
- 对每个关键词搜索 `app/`、`docs/`、`rules/`、`dev/`。
- 命中后必须失败，除非命中位置是当前任务说明或验收记录。
- 命中结果必须列出路径和行号。

### 6. Git 分支

实现任务必须检查：

```text
当前分支不是 main
当前分支不是 master
存在未提交文件时给出列表
失败分支不得合并
```

初始化任务可允许在 main 完成首次提交，但必须只包含初始化内容。

判定规则：

- `--task-type init --allow-main-init` 可在 `main` 或 `master`。
- 其他任务在 `main` 或 `master` 必须失败。
- 无法读取 Git 分支时，返回 exit code `2`。
- 有未提交文件不一定失败，但必须列出；PR 前必须清空或说明。

### 7. 文档入口

必须检查：

```text
根 AGENTS.md 存在
AGENTS.md 指向正式文档位置
docs/ 中存在 PRD、AI Coding 规范、测试验收规范
规则卡目录存在
```

必须存在：

```text
AGENTS.md
README.md
PRD_账房先生_本地财务Agent工作台.md
docs/00_文档索引.md
docs/01_项目文件地图与交付边界.md
docs/02_测试与验收规范.md
docs/03_AI_Coding_开发协作规范.md
rules/
```

### 8. Skill 目录

必须检查：

```text
skills/skill-creator/SKILL.md 存在
skills/index.json 存在
workspaces/default/skills/index.json 存在
不存在多余 Skill 中间层目录
不存在把 Skill 放入 data/
不存在按 Agent 单独拆分的根级 Skill 目录
```

失败时必须列出具体路径。

### 9. 模板一致性

当以下文件存在时必须检查：

```text
.github/ISSUE_TEMPLATE/task.md
.github/pull_request_template.md
dev/reports/browser_acceptance/README.md
```

唯一来源：

```text
docs/12_任务单与PR模板规范.md
docs/15_浏览器验收记录规范.md
docs/23_GitHub模板与验收记录落地规范.md
```

判定规则：

| 检查 | 失败条件 |
|---|---|
| `issue_template_consistency` | Issue 模板存在，但未标明来源为 `TASK CARD`，或字段标题与来源不一致 |
| `pr_template_consistency` | PR 模板存在，但未标明来源为 `PR TEMPLATE`，或字段标题与来源不一致 |
| `browser_acceptance_template_consistency` | 浏览器验收 README 存在，但未指向 `RECORD TEMPLATE`，或记录命名规则不一致 |
| `template_source_drift` | 目标模板含有来源中不存在的一级或二级标题 |

抽取规则：

- 从唯一来源中按固定小节标题抽取模板代码块。
- 只比较 Markdown 标题行和列表字段名。
- 不比较空行、说明文字、示例序号。
- 来源文件缺失时返回 exit code `2`。

不存在目标模板文件时，本检查为 `skipped`，不得失败。

## IMPLEMENTATION SHAPE

实现链路：

```text
GuardContext
-> CheckResult[]
-> 中文摘要
-> JSON 输出
-> exit code
```

核心函数：

```python
def build_context(args: argparse.Namespace) -> GuardContext:
    ...

def run_checks(context: GuardContext) -> list[CheckResult]:
    ...

def render_human(results: list[CheckResult]) -> str:
    ...

def render_json(results: list[CheckResult]) -> str:
    ...
```

`check_project_guard.py` 推荐结构：

```text
parse_args()
get_repo_root()
collect_git_status()
run_checks()
print_human_summary()
print_json()
exit_with_code()

check_root_allowlist()
check_github_boundary()
check_app_boundary()
check_docs_boundary()
check_dev_boundary()
check_data_boundary()
check_gitignore_required()
check_temp_artifacts()
check_parallel_implementations()
check_delete_residuals()
check_git_branch()
check_doc_entries()
check_skill_dirs()
check_template_consistency()
```

每个 `check_*` 只返回 `CheckResult`，不要在检查函数里直接退出程序。

`CheckResult` 字段：

```text
id
name
status
severity
messages
paths
```

## POWERSHELL ENTRY

`check_project_guard.ps1` 内容只允许做三件事：

```text
定位仓库根目录
调用 python dev/scripts/check_project_guard.py 并透传参数
返回 Python 脚本 exit code
```

不得在 PowerShell 入口里重复实现检查逻辑。

## SAMPLE OUTPUT FILE

必须生成：

```text
dev/reports/project_guard_sample_output.md
```

内容必须包含：

```text
一次通过输出
一次失败输出
失败示例使用的临时违规文件路径
违规文件已删除的确认
```

## EXIT CODE

```text
0 = 通过
1 = 失败
2 = 检查无法执行
```

## NEVER

- 不要为了通过护栏而删除检查项。
- 不要把失败项降级成警告。
- 不要用人工口头说明替代脚本结果。
- 不要只输出“通过”，必须列出检查项。

## DONE

护栏脚本实现完成必须满足：

- 能在本地运行。
- 成功时 exit code 为 0。
- 失败时 exit code 为 1。
- 输出中文摘要。
- 输出机器可读 JSON。
- 至少覆盖 REQUIRED CHECKS。
- 有最小测试或手动验证记录。
