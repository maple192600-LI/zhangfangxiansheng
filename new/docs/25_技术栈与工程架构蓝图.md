## ARCHITECTURE METHOD

所有产品代码按可测试链路设计：

```text
frontend view
-> frontend api module
-> backend api router
-> backend service
-> db model 或 core tool
-> audit / event / status record
```

实施方案不得只列页面和数据表，必须写清：

```text
参考依据
模块路径
核心接口
调用链
状态流转
权限边界
测试方法
浏览器验收
```

Agent、Skill、工作流、连接器、文件适配、报表生成相关模块，必须先写架构蓝图，再写里程碑实施方案。

## CHINESE INTERACTION LAYER

用户界面和用户命令使用中文，工程契约使用稳定英文 ID。

必须实现：

```text
CommandRegistry
CommandParser
CommandRouter
PermissionGuard
```

命令流：

```text
中文命令或按钮
-> command_id
-> 参数 schema 校验
-> 权限检查
-> service / core tool
-> 中文结果
```

示例：

| 用户可见命令 | command_id | handler |
|---|---|---|
| `/压缩上下文` | agent.compact | AgentCommandService.compact |
| `/查看状态` | agent.status | AgentCommandService.status |
| `/运行工作流` | workflow.run | WorkflowService.execute |
| `/生成报表` | report.generate | ReportGenerationService.generate |

不得让业务逻辑依赖中文文案。中文文案可以改，`command_id`、API、tool name 不随文案变化。

## EXCEL AND TABLE ADAPTERS

内置默认：

| 能力 | 默认工具 | 用途 |
|---|---|---|
| 读取 `.xlsx` 工作簿、sheet、单元格、基础样式 | openpyxl | Excel 预览、模板读取、基础写入 |
| 表格清洗、筛选、聚合 | Polars | 结构化数据处理 |
| 本地多表分析查询 | DuckDB | 大表查询、报表取数、临时分析 |

可选扩展：

| 能力 | 可选工具 | 触发方式 |
|---|---|---|
| pandas 生态兼容、部分第三方项目依赖 | pandas | Skill 声明依赖 |
| 新建高格式 Excel、图表和样式 | xlsxwriter | Skill 或报表节点声明依赖 |
| 高保真转换、复杂 Excel/PDF 渲染 | LibreOffice | 用户配置路径后启用 |

选择规则：

```text
基础 xlsx 读写优先 openpyxl
大批量结构化处理优先 Polars
多表查询和分析优先 DuckDB
复杂新建报表格式可选 xlsxwriter
第三方 Skill 需要 pandas 时由 Skill 声明
高保真转换才调用 LibreOffice
```

不得把某个 Excel 样本绑定到唯一库。库选择属于工具适配层或 Skill 能力声明，不写进业务规则。

## BACKEND LAYOUT

```text
app/backend/
  main.py
  config.py
  database.py
  api/
    health.py
  services/
  db/
    models/
  schemas/
  core/
    responses.py
    errors.py
    logging.py
    files/
    excel/
    agent/
    skills/
    workflows/
    connectors/
  tests/
```

职责：

| 路径 | 职责 |
|---|---|
| `main.py` | 创建 FastAPI app、注册路由、挂载静态文件 |
| `config.py` | 读取项目路径、环境变量、数据目录 |
| `database.py` | SQLite engine、session、初始化入口 |
| `api/` | HTTP 路由，只做请求响应转换 |
| `services/` | 业务用例编排 |
| `db/models/` | SQLAlchemy 模型 |
| `schemas/` | Pydantic 请求响应结构 |
| `core/responses.py` | 统一响应格式 |
| `core/errors.py` | 错误码和异常转换 |
| `core/files/` | 文件仓库、hash、留存策略 |
| `core/excel/` | Excel 预览、读取、写入 |
| `core/agent/` | 会话、消息、上下文、工具调用 |
| `core/skills/` | Skill 扫描、登记、启用、测试 |
| `core/workflows/` | 工作流配方、节点、执行记录 |
| `core/connectors/` | CLI、API、外部系统连接器 |
| `tests/` | 后端测试 |

## API RULE

所有后端接口返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

前端不得绕过 `src/api/client.ts` 直接解析接口。

## LAYER RULE

```text
frontend view -> frontend api client -> backend api -> service -> db/core tool
```

禁止：

```text
api 直接写复杂业务逻辑
frontend 直接拼后端数据口径
Agent 直接写正式数据库
Skill 直接绕过权限执行高风险动作
工作流节点绕过执行记录
```
