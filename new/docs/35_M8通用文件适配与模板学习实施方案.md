## FILE LIFECYCLE

上传文件不得无治理永久堆积。

每个文件必须记录：

```text
file_id
source
purpose
content_hash
size_bytes
storage_path
retention_policy
linked_object_ids
linked_rule_ids
status
created_at
expires_at
```

规则学习过程使用的样本文件必须可追溯，但可按留存策略归档或清理。清理前必须确认没有启用规则、工作流、报表绑定依赖该文件。

## TOOL OUTPUT BUDGET

Agent 读取文件时只返回预览和摘要：

```text
表头候选
前 N 行样例
字段类型分布
异常摘要
结构指纹
```

不得把整份文件内容塞入 Agent 上下文。大输出必须写入工作区产物，由事件引用。

## FLOW

```text
1. 文件进入文件仓库
2. 生成结构指纹
3. 匹配已有规则
4. 高置信命中则生成预览
5. 未命中或低置信进入学习流程
6. Agent 生成规则草稿
7. 工具层样本试跑
8. 用户查看差异和异常
9. 用户确认启用
10. 同类文件再次上传复用规则
```

## FINGERPRINT RULE

指纹不得依赖：

```text
文件名
固定绝对路径
固定 Sheet 名
固定列号
```

指纹应包含：

```text
文件类型
sheet 数量
候选表头文本
列数
字段类型分布
样例值模式
空行和合并单元格摘要
```

## TESTS

```text
同结构不同文件名命中同一规则
文件名相同但结构不同不误命中
低置信进入学习流程
规则草稿不能直接启用
样本试跑失败不能启用
启用后同类文件复用规则
回滚后使用指定版本
清理样本文件前能检测依赖关系
Agent 上下文只包含文件摘要和产物引用
```

## BROWSER ACCEPTANCE

```text
上传已知结构文件
系统命中规则并生成预览
上传未知结构文件
页面进入学习流程
Agent 生成规则草稿
运行样本试跑
用户确认启用
再次上传同类文件自动命中
格式变化时进入规则修正
```
