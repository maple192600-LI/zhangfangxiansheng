# 11_正式产品删除与数据生命周期规范

## READ WHEN

设计或实现正式产品中的删除、撤销、作废、停用、归档、清理、恢复、彻底删除时必须读取。

本规范只适用于产品运行后的用户数据和业务配置，不适用于开发阶段测试功能的删除。

## CORE RULE

产品中的“删除”必须有明确语义。不能只从前端列表移除，也不能悄悄保留后端残留。

用户点击删除前，系统必须说明：

```text
删除对象是什么
是否会删除文件本体
是否会删除数据库记录
是否会影响已生成报表
是否会影响工作流 / 规则 / Skill
是否可恢复
是否会留下审计记录
```

## ACTION TYPES

| 操作 | 含义 | 适用对象 |
|---|---|---|
| 取消 | 任务尚未确认，直接退出流程 | 上传中、导入预览、规则草稿 |
| 删除草稿 | 删除未确认内容和临时产物 | 模板草稿、规则草稿、工作流草稿 |
| 撤销导入 | 回滚一次未锁定导入批次 | 文件导入批次、解析结果 |
| 作废 | 保留记录，但业务上不再有效 | 已确认流水、报表生成记录 |
| 停用 | 保留配置，但不允许后续使用 | 账户、模板、规则、Skill、工作流 |
| 归档 | 保留记录，默认视图不展示 | 文件、报表、日志、低频数据 |
| 清理 | 删除可重建缓存和临时产物 | 预览缓存、导出缓存、运行日志 |
| 彻底删除 | 删除对象、文件、元数据和关联草稿 | 未被正式引用的文件、草稿、测试对象 |

## OBJECT RULES

### 文件

文件删除必须区分：

```text
只删除文件本体，保留元数据和入库数据
删除文件本体 + 元数据
删除文件本体 + 元数据 + 解析结果
删除导入批次并回滚结构化数据
```

已参与确认入库、报表生成或审计记录的文件，默认不能静默彻底删除。

### 模板 / 规则 / Skill

删除或停用前必须检查：

```text
是否被工作流引用
是否被报表模板引用
是否被 Agent 配置引用
是否被测试样本引用
是否有启用中的版本记录
```

存在引用时，必须先迁移引用、停用，或提示用户不能直接彻底删除。

### 工作流

已启用工作流不得直接物理删除。可选操作：

```text
停用
归档
复制后调整
迁移引用后删除
```

草稿工作流可以彻底删除。

### 业务数据

业务数据按状态处理：

```text
草稿：可删除
未确认导入：可撤销或删除
已确认入库：作废 / 撤销导入 / 反向调整
已参与报表：保留来源记录，按影响范围处理
主数据：停用 / 合并 / 迁移引用后删除
```

### 报表

报表删除必须区分：

```text
删除导出文件
删除生成记录
删除报表模板绑定
删除报表模板
```

删除导出文件不得破坏生成记录和取数来源记录。

## REQUIRED IMPACT CHECK

执行前必须检查：

```text
前端引用
后端 API
数据库记录
文件仓库
解析结果
报表记录
工作流引用
规则引用
Skill 引用
Agent 配置引用
审计记录
备份 / 恢复点
```

## USER CONFIRMATION

以下操作必须二次确认：

- 彻底删除。
- 撤销导入批次。
- 删除文件本体。
- 删除模板 / 规则 / Skill。
- 停用被工作流引用的对象。
- 影响已生成报表的数据操作。

确认弹窗必须用中文说明影响范围，不得只显示“是否删除”。

## NEVER

- 不要只删除前端列表项。
- 不要只隐藏按钮。
- 不要只删除文件本体而保留不可见元数据。
- 不要只删除数据库记录而保留文件仓库内容。
- 不要删除对象后让工作流、报表、规则继续引用不存在的对象。
- 不要把“停用”“归档”“作废”伪装成“删除”。
- 不要在未说明影响范围时执行彻底删除。

## REQUIRED OUTPUT

涉及产品删除链路的任务完成时必须输出：

```text
操作语义：
影响对象：
前端处理：
后端处理：
数据库处理：
文件仓库处理：
规则 / 工作流 / Skill 引用处理：
审计记录：
是否可恢复：
浏览器验收：
```

## BROWSER CHECK

必须验证：

- 删除 / 停用 / 作废入口文案准确。
- 确认弹窗显示影响范围。
- 操作后列表状态正确。
- 刷新页面后状态仍正确。
- 相关报表、工作流、规则不引用失效对象。
- 错误提示中文可读。
