## REFERENCE BASIS

| 参考对象 | 采用能力 | 本方案落点 |
|---|---|---|
| Ant Design Vue 后台布局模式 | 侧边导航、顶部状态区、内容容器 | `MainLayout.vue`、导航菜单、状态栏 |
| Vue Router | 路由表、刷新恢复、页面入口 | `router/index.ts` |
| 财务软件操作习惯 | 表格优先、入口清晰、少装饰 | 首页、模块页、状态区 |
| Claude Code / Codex 浏览器验收方式 | 可点击、可刷新、可复查 | 浏览器验收记录 |

不采用：

```text
不采用营销页结构
不采用无 API 支撑的假业务统计
不采用把 Agent 管理塞进普通业务页面
```

## ARCHITECTURE

M3 只实现页面壳和入口体系：

```text
App.vue
-> MainLayout.vue
-> router/index.ts
-> views/*
-> components/PagePlaceholder.vue
-> components/AgentEntryButton.vue
-> components/AppShellStatus.vue
```

所有未实现业务闭环的页面必须使用统一占位组件。

## CORE INTERFACES

路由记录结构：

```ts
type ModuleRoute = {
  path: string
  name: string
  component: Component
  meta: {
    title: string
    module: string
  }
}
```

占位组件入参：

```ts
type PagePlaceholderProps = {
  title: string
  purpose: string
  milestone: string
  availableActions: string[]
}
```

Agent 入口占位事件：

```ts
function handleAgentClick(sourcePage: string): void
```

## FLOW

```text
1. 用户打开首页
2. MainLayout 渲染导航和状态区
3. AppShellStatus 请求 /api/health
4. 用户点击导航
5. Vue Router 切换页面
6. 未实现页面展示 PagePlaceholder
7. AgentEntryButton 点击后显示未启用提示
8. 刷新页面后当前路由恢复
```

## NAVIGATION

左侧主导航顺序必须为：

```text
首页
日常工作台
文件导入
报表中心
数据中心
工作流中心
Agent 中心
规则与自动化
连接器
文件仓库
系统管理
```

路由：

| 导航 | 路由 | 页面 |
|---|---|---|
| 首页 | `/` | `HomeView.vue` |
| 日常工作台 | `/daily-workbench` | `DailyWorkbenchView.vue` |
| 文件导入 | `/files/import` | `FileImportView.vue` |
| 报表中心 | `/reports` | `ReportCenterView.vue` |
| 数据中心 | `/data/objects` | `DataObjectsView.vue` |
| 工作流中心 | `/workflows` | `WorkflowCenterView.vue` |
| Agent 中心 | `/agents` | `AgentCenterView.vue` |
| 规则与自动化 | `/rules` | `RulesAutomationView.vue` |
| 连接器 | `/connectors` | `ConnectorsView.vue` |
| 文件仓库 | `/files/repository` | `FileRepositoryView.vue` |
| 系统管理 | `/settings` | `SystemManagementView.vue` |

`/data/objects` 复用 M2 的 `DataObjectsView.vue`。

## HOME VIEW

首页必须显示：

```text
今日工作
待处理事项
最近导入
最近报表
Agent 任务
异常提醒
快捷入口
```

M3 不实现真实业务统计。

首页卡片数据来源：

```text
后端健康状态
静态入口配置
空状态提示
```

不得写假金额、假报表数量、假异常数量。

## PLACEHOLDER PAGE RULE

未实现业务闭环的页面必须使用 `PagePlaceholder.vue`。

占位页必须显示：

```text
页面名称
页面用途
当前状态：待实现
后续里程碑
可用入口
```

不得显示虚假业务数据。

## AGENT ENTRY PLACEHOLDER

`AgentEntryButton.vue` 是入口占位，不实现 Agent 对话。

点击后显示中文提示：

```text
Agent 能力将在后续里程碑启用
```

不得调用不存在的 Agent API。

## STATUS AREA

`AppShellStatus.vue` 必须显示：

```text
后端状态
当前模式
项目护栏提示
```

后端状态来自：

```text
GET /api/health
```

项目护栏提示在 M3 只显示静态文案：

```text
提交前请运行项目护栏检查
```

不得假装已经运行护栏。

## VISUAL STYLE

必须使用 CSS 变量：

```text
--zf-bg
--zf-surface
--zf-border
--zf-text
--zf-muted
--zf-primary
--zf-primary-soft
--zf-accent
```

页面风格：

```text
左侧 280px 导航
顶部 56px 状态区
内容区最小宽度适配
表格和列表优先
卡片只用于入口和摘要
中文 UI
```

不得使用：

```text
大面积渐变
营销 hero
装饰性漂浮图形
假图表
过度圆角
```

## API

M3 不新增后端 API。

只允许使用：

```text
GET /api/health
M2 已存在的对象 API
```

新增按钮如果没有 API，必须明确是占位并显示中文提示。

## TESTS

前端必须通过：

```text
cd app/frontend
npm run typecheck
npm run build
```

如果已有前端测试框架，可增加路由渲染测试；没有测试框架时不得为了 M3 临时引入复杂测试栈。

## BROWSER ACCEPTANCE

验收记录：

```text
dev/reports/browser_acceptance/YYYYMMDD_HHMM_m3_ui_shell.md
```

必须验证：

```text
首页可打开
左侧导航顺序正确
每个导航入口可点击
刷新后当前路由不丢
状态区显示后端状态
Agent 入口点击后显示未启用提示
数据中心入口能进入 /data/objects
浏览器控制台无关键错误
```
