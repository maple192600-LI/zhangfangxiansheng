# 12_任务单与PR模板规范

## READ WHEN

开始开发任务、拆分任务、创建 GitHub Issue、提交 PR、阶段验收前必须读取。

## PURPOSE

任务单用于开工前锁定目标。PR 用于合并前验收。

AI Coding 工具不得拿到一句用户指令就直接改代码。必须先把指令转成任务单，并在完成时按 PR 模板交付。

## SPLIT RULES

必须拆分任务的情况：

- 同时改多个独立功能域。
- 同时改前端界面和底层数据模型，且影响范围不清楚。
- 同时新增功能和删除功能。
- 同时做产品功能和开发环境调整。
- 一次任务无法在浏览器中清晰验收。

## NEVER

- 不要无任务单直接改代码。
- 不要把多个无关需求塞进一个 PR。
- 不要在 PR 中只写“已完成”。
- 不要省略浏览器验收。
- 不要省略影响检查。
- 不要把失败项写进“已知问题”后继续合并。

## DONE

任务完成必须满足：

- 任务单目标已覆盖。
- 不做范围未被越界修改。
- PR 模板字段完整。
- 浏览器验收可复查。
- 项目护栏检查通过。
- 用户检查路径明确。
