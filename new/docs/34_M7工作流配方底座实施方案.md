## DETERMINISTIC VALIDATION

工作流草稿启用前必须通过确定性校验：

```text
has_nodes
has_trigger
all_nodes_connected
no_unreachable_nodes
valid_required_parameters
valid_options_values
expressions_reference_existing_nodes
tools_have_parameters
no_hardcoded_credentials
no_unnecessary_code_nodes
```

校验失败时必须返回中文定位：

```text
失败节点
失败字段
失败原因
建议修复动作
```

## FLOW

```text
1. Agent 或用户创建配方草稿
2. RecipeValidator 校验 schema
3. RecipeValidator 运行确定性校验
4. NodeRegistry 检查节点类型
5. 页面显示步骤卡片
6. 高级用户查看画布
7. SampleRunner 执行样本
8. 每个节点写 node_event
9. 样本通过后等待用户确认
10. 用户确认启用版本
11. 后续运行由 WorkflowExecutor 确定性执行
```

## TESTS

```text
创建草稿成功
未知节点类型校验失败
节点输入输出不匹配校验失败
样本试跑写 node_events
失败节点可定位
未样本通过不能启用
启用版本不能覆盖
执行已启用配方不调用 Agent 推理
断开的节点校验失败
引用不存在节点输出校验失败
缺少必填参数校验失败
```

## BROWSER ACCEPTANCE

```text
打开 /workflows
创建一个草稿配方
查看步骤卡片
进入画布查看节点和连线
运行样本试跑
失败时看到失败节点
成功后点击启用
刷新后版本记录仍存在
再次执行配方不依赖 Agent 重新推理
```
