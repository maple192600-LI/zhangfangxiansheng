# 17_数据模型与字段字典设计规范

## READ WHEN

设计或修改数据库表、字段、主数据、动态业务对象、导入映射、页面筛选项、报表字段、工作流字段引用时必须读取。

## CORE RULE

产品中的名称不是事实来源。稳定 `id` 才是事实来源。

页面展示名称可以改；关联关系必须使用稳定 `id`。

## IMPLEMENTATION METHOD

数据模型必须按三层实现：

```text
定义层：dynamic_objects / field_definitions
数据层：dynamic_object_rows 或对应业务表
引用层：规则、工作流、报表、Skill、Agent 配置只引用稳定 id 或 field_key
```

写入链路：

```text
前端表单
-> API
-> service 校验唯一性和引用
-> db model
-> audit_logs
-> 前端刷新
```

核心接口：

```python
def resolve_field(object_id: str, field_key: str) -> FieldDefinition:
    ...

def rename_display_name(target_type: str, target_id: str, display_name: str) -> None:
    ...

def find_reference_impact(target_type: str, target_id: str) -> ImpactReport:
    ...
```

## MUST

- 主数据必须有稳定 `id`。
- 字段定义必须进入字段字典。
- 页面、报表、工作流、规则、Skill 引用字段时，必须引用字段 `key` 或字段 `id`，不得引用展示名。
- 账户、组织、部门、项目、客户、供应商、报表模板、工作流、规则、Skill 必须支持唯一标识。
- 用户改名称时，所有展示位置自动显示调整后的名称。
- 用户改归属关系时，相关列表、筛选、报表、工作流必须同步或提示影响范围。
- 需要保留口径时，保存快照字段，不要把快照和当前主数据混用。

## FIELD DICTIONARY

字段字典至少包含：

```text
field_id
object_id
field_key
display_name
data_type
required
default_value
enum_source
relation_object
relation_field
is_system
is_active
created_at
updated_at
```

## MASTER DATA

主数据至少包含：

```text
id
code
name
status
parent_id
effective_from
effective_to
created_at
updated_at
```

不得用 `name` 作为主键或跨表关联依据。

## DYNAMIC OBJECT

动态业务对象至少包含：

```text
object_id
object_key
display_name
description
owner_agent_id
status
schema_version
created_at
updated_at
```

动态对象的数据行必须能追溯：

```text
row_id
object_id
source_file_id
import_batch_id
created_by
created_at
updated_at
```

## DISPLAY RULE

页面展示时：

```text
存 id
查 id
展示 name
导出时按当前展示名或指定快照名
```

如果报表需要保留当时口径：

```text
保存 snapshot_name
保存 snapshot_code
保存 snapshot_time
```

## IMPACT CHECK

修改主数据或字段定义前必须检查：

```text
列表页
详情页
筛选项
导入映射
报表模板
工作流配方
规则
Skill
Agent 配置
导出文件
测试样本
```

影响检查必须返回结构化结果：

```json
{
  "can_change": true,
  "affected_pages": [],
  "affected_reports": [],
  "affected_workflows": [],
  "affected_rules": [],
  "warnings": []
}
```

## NEVER

- 不要用名称做关联键。
- 不要把字段展示名写死在业务逻辑里。
- 不要在多个页面各自维护一份主数据名称。
- 不要修改设置后只让当前页面生效。
- 不要让工作流保存易变展示名作为字段引用。
- 不要把快照口径和当前主数据混用。

## BROWSER CHECK

涉及主数据或字段字典时必须验证：

- 修改名称后，相关列表展示同步。
- 修改归属后，筛选项和详情页同步。
- 报表预览使用正确名称。
- 导出结果使用正确名称或快照名称。
- 工作流引用仍然有效。
- 刷新页面后仍然正确。

## DONE

```text
稳定 id：
字段字典：
关联方式：
快照口径：
影响检查：
浏览器验收：
```
