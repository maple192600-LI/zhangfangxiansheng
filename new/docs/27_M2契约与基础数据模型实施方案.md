## REFERENCE BASIS

| 参考对象 | 采用能力 | 本方案落点 |
|---|---|---|
| SQLAlchemy 2.x | ORM 模型、事务 session、service 层数据操作 | `db/models/system.py`、`database.py`、`services/object_service.py` |
| Alembic | 可追踪数据库结构变更 | `alembic/`、迁移命令 |
| FastAPI + Pydantic | 请求校验、响应结构、错误转换 | `schemas/system.py`、`api/objects.py` |
| 财务主数据软件常见模式 | 稳定 id、展示名可变、字段字典 | `dynamic_objects`、`field_definitions` |

不采用：

```text
不采用展示名作为主键
不采用前端生成正式 id
不采用绕过 service 的路由直接写库
```

## ARCHITECTURE

M2 建立对象与字段的单一事实源：

```text
frontend view
-> app/frontend/src/api/objects.ts
-> backend api/objects.py
-> services/object_service.py
-> db/models/system.py
-> SQLite
```

所有创建、修改和校验必须经过 service。

## CORE INTERFACES

service 层至少提供：

```python
def list_objects(session: Session) -> list[DynamicObject]:
    ...

def create_object(session: Session, payload: ObjectCreate) -> DynamicObject:
    ...

def list_fields(session: Session, object_id: str) -> list[FieldDefinition]:
    ...

def create_field(session: Session, object_id: str, payload: FieldCreate) -> FieldDefinition:
    ...

def update_field(session: Session, field_id: str, payload: FieldUpdate) -> FieldDefinition:
    ...
```

前端 API 至少提供：

```ts
export function listObjects(): Promise<DynamicObject[]>
export function createObject(input: CreateObjectInput): Promise<DynamicObject>
export function listFields(objectId: string): Promise<FieldDefinition[]>
export function createField(objectId: string, input: CreateFieldInput): Promise<FieldDefinition>
export function updateField(fieldId: string, input: UpdateFieldInput): Promise<FieldDefinition>
```

## FLOW

```text
1. 用户在 /data/objects 创建对象
2. 前端调用 createObject
3. API 层只转发请求
4. service 校验 object_key 唯一
5. 后端生成稳定 id
6. 写 dynamic_objects
7. 写 audit_logs
8. 返回统一响应
9. 前端刷新对象列表
10. 字段创建和字段修改走同样 service 链路
```

## TABLES

### dynamic_objects

| 字段 | 类型 | 约束 |
|---|---|---|
| id | string | 主键，稳定 id |
| object_key | string | 唯一，不使用展示名 |
| display_name | string | 可修改 |
| description | string | 可空 |
| status | string | active / disabled / archived |
| schema_version | integer | 默认 1 |
| created_at | datetime | 必填 |
| updated_at | datetime | 必填 |

### field_definitions

| 字段 | 类型 | 约束 |
|---|---|---|
| id | string | 主键，稳定 id |
| object_id | string | 外键，指向 dynamic_objects.id |
| field_key | string | 同一 object 内唯一 |
| display_name | string | 可修改 |
| data_type | string | text / number / decimal / date / datetime / boolean / enum / relation |
| required | boolean | 默认 false |
| default_value | string | 可空 |
| enum_source | string | 可空 |
| relation_object | string | 可空 |
| relation_field | string | 可空 |
| is_system | boolean | 默认 false |
| is_active | boolean | 默认 true |
| created_at | datetime | 必填 |
| updated_at | datetime | 必填 |

### audit_logs

| 字段 | 类型 | 约束 |
|---|---|---|
| id | string | 主键 |
| actor | string | 默认 local_user |
| action | string | 必填 |
| target_type | string | 必填 |
| target_id | string | 可空 |
| summary | string | 中文摘要 |
| payload_json | text | JSON 字符串 |
| created_at | datetime | 必填 |

## ID RULE

```text
obj_<uuid>

```

前端不得生成正式 id。

## API

### GET /api/objects

返回动态业务对象列表。

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": []
  }
}
```

### POST /api/objects

创建动态业务对象。

请求：

```json
{
  "object_key": "project",
  "display_name": "项目台账",
  "description": "项目基础信息"
}
```

响应返回创建后的对象。

### GET /api/objects/{object_id}/fields

返回字段定义列表。

### POST /api/objects/{object_id}/fields

创建字段定义。

请求：

```json
{
  "field_key": "project_name",
  "display_name": "项目名称",
  "data_type": "text",
  "required": true
}
```

### PATCH /api/fields/{field_id}

修改字段展示名、必填、启用状态等。

不得修改：

```text
id
object_id
field_key
created_at
```

## SERVICE RULE

`api/objects.py` 只做：

```text
接收请求
调用 service
返回统一响应
```

`services/object_service.py` 负责：

```text
校验 object_key 唯一
校验 field_key 唯一
创建对象
创建字段
修改字段
写 audit_logs
```
