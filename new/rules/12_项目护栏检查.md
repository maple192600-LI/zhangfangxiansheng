# 12_项目护栏检查

## READ WHEN

任务完成前、创建 PR 前、合并前、打包前读取。

## READ ALSO

`../docs/08_项目护栏检查规范.md`

## MUST

- 运行 `dev/scripts/check_project_guard`。
- 检查文件边界、临时产物、环境隔离、替换实现清理、GitHub 分支状态。
- 护栏失败先修复，不得继续合并。
- 完成说明写出护栏检查结论。

## NEVER

- 不要删除护栏检查项来通过任务。
- 不要把护栏失败当成普通已知问题。
- 不要用人工口头说明替代检查结果。

## CHECK

```text
项目护栏检查：通过 / 失败 / 尚未实现，不得声明完成
```
