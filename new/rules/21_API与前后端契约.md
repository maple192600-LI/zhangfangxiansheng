# 21_API与前后端契约

## READ WHEN

设计或修改页面、按钮、接口、错误码、前端 API 调用、后端路由、数据刷新、上传、导出、删除动作时读取。

## READ ALSO

`../docs/18_API与前后端契约规范.md`

## MUST

- 每个用户动作有明确前后端契约。
- 新增按钮定义 API 或明确只做前端状态。
- 新增 API 必须有页面或工作流使用方。
- 修改 API 同步修改前端调用、测试、文档。
- 删除功能清理页面调用、API、service、测试和契约。
- 错误信息中文可读。
- 状态必须能刷新后恢复。

## NEVER

- 不要只改前端不改 API 契约。
- 不要只加后端接口不接页面。
- 不要前端写死成功数据。
- 不要吞掉后端错误。
- 不要保留无人使用的 API。

## DONE

```text
API 契约：
页面 API 映射：
状态流转：
错误码：
浏览器验收：
```
