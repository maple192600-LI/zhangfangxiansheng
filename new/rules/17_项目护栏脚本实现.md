# 17_项目护栏脚本实现

## READ WHEN

实现、修改、运行 `dev/scripts/check_project_guard` 时读取。

## READ ALSO

`../docs/14_项目护栏检查脚本规格.md`

## MUST

- 护栏脚本必须可在本地运行。
- 输出中文摘要和 JSON。
- 支持 `--task-type`、`--allow-main-init`、`--delete-keywords`、`--json-only`。
- 检查文件边界、提交排除、临时产物、并行替代实现、删除残留、Git 分支、文档入口、Skill 目录、模板一致性。
- 失败时列出具体路径和原因。
- JSON 必须包含 `ok`、`task_type`、`branch`、`checks`、`summary`。
- 每个检查项必须包含 `id`、`name`、`status`、`severity`、`messages`、`paths`。
- PowerShell 入口只调用 Python 脚本并透传参数。
- 失败 exit code 为 1。
- 无法执行 exit code 为 2。

## NEVER

- 不要删除检查项来通过。
- 不要把失败降级成普通提示。
- 不要只输出“通过”。
- 不要用人工说明替代脚本结果。
- 不要在 PowerShell 入口里重复实现检查逻辑。
- 不要只检查根目录而忽略 `app/`、`docs/`、`dev/`、`data/`、`skills/`、`workspaces/`。
- 不要让 `.github/` 模板和 `dev/reports/browser_acceptance/README.md` 自成一套字段。

```text
命令：
exit code：
中文摘要：

失败样例：
PowerShell 入口：
样例输出文件：
模板一致性：
```
