# 08_临时文件与开发产物清理

## READ WHEN

任务会生成导出文件、下载文件、截图、日志、临时数据库、调试脚本、测试报告时读取。

## READ ALSO

`../docs/05_开发产物与清理规范.md`

## MUST

- 任务开始前说明会产生哪些临时产物。
- 临时文件写入 `data/tmp/` 或 `data/downloads/`。
- 测试报告摘要写入 `dev/reports/`。
- 用户确认的测试样本写入 `dev/fixtures/`。
- 完成前删除可重建临时文件。
- 完成说明写清楚清理结果。

## NEVER

- 不要把临时文件留在仓库根目录。
- 不要把测试下载文件提交到 GitHub。
- 不要删除用户主动提供的测试样本。
- 不要让失败试跑文件留在产品目录。

## CHECK

```text
临时文件：已清理 / 无临时文件 / 未清理并说明原因
测试样本：新增路径 / 未新增
运行产物：未进入 GitHub 提交
```
