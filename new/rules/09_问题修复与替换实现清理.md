# 09_问题修复与替换实现清理

## READ WHEN

修 bug、替换实现、重构、接口调整、页面调整、解析器修复、工作流修复时读取。

## READ ALSO

`../docs/06_问题修复与代码替换规范.md`

## MUST

- 先复现问题。
- 定位真实出错链路。
- 列出受影响页面、API、服务、数据表、工作流、测试。
- 修改原实现。
- 同步修改所有调用方。
- 删除被替换的原入口和原文件。
- 搜索原名称、原路径、原接口，残留必须为 0。

## NEVER

- 不要创建并行替代文件。
- 不要新建相似接口绕开原接口。
- 不要新建相似页面绕开原页面。
- 不要只修当前报错点，不查调用链。
- 不要降低校验或改测试掩盖错误。

## CHECK

- 原文件已删除。
- 原路由已删除。
- 原菜单已删除。
- 原 API 调用已删除。
- 原测试已删除或改为验证替代实现。
- 浏览器验收通过。
