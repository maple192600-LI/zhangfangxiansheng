# 02_单一事实源与数据联动

## READ WHEN

任务涉及账户、组织、部门、项目、客户、供应商、科目、收支分类、字段字典、系统设置、数据中心时必须读取。

MUST:

- 主数据使用稳定 `id` 关联。
- 页面显示名通过统一查询层解析。
- 修改主数据后，相关页面刷新、重算或提示受影响范围。
- 报表需要保留记录口径时，明确保存快照字段。
- 当前值和口径快照必须区分。

NEVER:

- 用名称文本做主键。
- 在多个页面各存一份账户名称或组织名称。
- 只让设置页生效，其他页面不变。
- 绕过字段字典直接写死显示名。

## TEST

至少验证一种：

```text
修改账户名称
修改所属核算组织
修改部门名称
修改字段字典显示名
修改收支分类
```

然后回到相关页面验证：

- 列表显示。
- 筛选条件。
- 报表预览。
- 导出结果。
- 工作流取数。

## CHECK

- 是否所有关联都用 `id`？
- 是否有统一查询或解析层？
- 是否说明口径快照和当前值？
- 是否做过跨页面联动验收？
