# 19_样本与测试文件管理

## READ WHEN

接收用户文件、新增样本、修改预期输出、删除样本、做浏览器上传验收时读取。

## READ ALSO

`../docs/16_样本与测试文件管理规范.md`

## MUST

- 用户确认的测试样本放 `dev/fixtures/`。
- 临时导出、下载、截图放 `data/`。
- 新增样本更新 `dev/fixtures/manifest.md`。
- 修改 `expected/` 必须说明需求原因。
- 删除样本前检查测试、浏览器验收、规则、工作流引用。

## NEVER

- 不要把样本放进 `app/`。
- 不要把临时导出当作预期输出。
- 不要为通过测试而修改 `expected/`。
- 不要用文件名驱动业务逻辑。
- 不要无说明删除用户样本。

## DONE

```text
新增样本：
修改样本：
删除样本：
manifest 更新：
临时产物：
```
