# 11_GitHub分支与PR开发

## READ WHEN

创建仓库、创建分支、提交、推送、PR、合并、删除失败分支、多设备继续开发时读取。

## READ ALSO

`../docs/07_GitHub私有仓库与分支PR规范.md`

## MUST

- 仓库使用 Private。
- 从 `main` 拉取远端当前代码后再开任务分支。
- 每个任务一个分支。
- 本地运行和浏览器验收通过后再提交 PR。
- PR 写清修改范围、测试结果、浏览器验收、风险、用户检查路径。
- 失败任务分支必须删除。

## NEVER

- 不要直接在 `main` 开发。
- 不要把失败分支合并到 `main`。
- 不要把失败代码长期留在分支里。
- 不要把 `data/`、`.venv/`、`node_modules/` 推送到 GitHub。

## CHECK

- 当前分支不是 `main`。
- PR 有浏览器验收记录。
- 失败分支已删除或无失败分支。
- 多设备开发时，开始前已同步远端当前代码。
