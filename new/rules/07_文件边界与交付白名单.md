# 07_文件边界与交付白名单

## READ WHEN

新增目录、移动文件、写测试样本、写脚本、打包交付、整理项目结构时读取。

## READ ALSO

`../docs/01_项目文件地图与交付边界.md`

## MUST

- 应用源码只放 `app/`。
- 开发文档只放 `docs/`。
- 测试样本只放 `dev/fixtures/`。
- 开发脚本只放 `dev/scripts/`。
- 运行产物只放 `data/`。
- 新顶层目录必须先登记到文件地图。
- 用户交付包使用白名单，不用全仓库打包。

## NEVER

- 不要把 PRD、测试样本、运行数据库放进 `app/`。
- 不要在仓库根目录随手放临时文件。
- 不要把 `docs/`、`dev/`、`data/` 打进用户交付包。

## CHECK

- 新文件是否放在正确目录？
- 是否新增未登记目录？
- 交付包是否排除开发资料和运行产物？
