# 22_工作流配方详细设计

## READ WHEN

设计或实现工作流中心、步骤卡片、画布节点、工作流草稿、样本试跑、启用、停用、回滚、执行记录、节点注册时读取。

## READ ALSO

`../docs/19_工作流配方详细设计规范.md`

## MUST

- 工作流配方是结构化数据。
- Agent 生成草稿，工具层确定性执行，用户确认后启用。
- 已启用配方再次运行不依赖 Agent 再次推理。
- 节点必须有输入、输出、参数、权限、错误处理。
- 启用前必须样本试跑。
- 执行失败必须定位到节点。
- 配方变更生成后续版本，不覆盖已启用版本。
- 工作流实施方案必须写清 NodeRegistry、SampleRunner、WorkflowExecutor 和执行记录。

## METHOD

```text
Agent / 页面生成草稿
-> RecipeValidator 校验
-> NodeRegistry 检查节点
-> SampleRunner 样本试跑
-> 用户确认
-> VersionManager 启用
-> WorkflowExecutor 确定性执行
-> WorkflowEventStore 记录节点事件
```

## NEVER

- 不要把工作流写死在按钮事件里。
- 不要让 Agent 每次临场决定正式报表逻辑。
- 不要跳过样本试跑。
- 不要让普通用户必须学画布。
- 不要让节点输入输出靠自然语言猜测。

## DONE

```text
配方 schema：
节点契约：
NodeRegistry：
样本试跑：
执行器：
执行记录：
版本规则：
浏览器验收：
```
