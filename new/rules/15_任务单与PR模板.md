# 15_任务单与PR模板

## READ WHEN

开始开发任务、拆分任务、创建 Issue、提交 PR、阶段验收前读取。

## READ ALSO

`../docs/12_任务单与PR模板规范.md`

## MUST

- 开工前输出任务单。
- 写明用户原始要求、本次目标、不做范围、必读文档、规则卡、修改对象、影响检查、验收计划。
- PR 写明目标、修改范围、删除 / 替换 / 生命周期、数据影响、验证结果、用户检查路径、风险、回退方式。
- 大任务必须拆分。
- PR 合并前必须有浏览器验收和项目护栏检查。

## NEVER

- 不要无任务单直接改代码。
- 不要把多个无关需求塞进一个 PR。
- 不要只写“已完成”。
- 不要省略影响检查。
- 不要省略浏览器验收。

## DONE

```text
任务单：
PR 模板：
浏览器验收：
项目护栏检查：
用户检查路径：
```
