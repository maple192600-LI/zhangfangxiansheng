# 06_PR与完成定义

## READ WHEN

任务接近完成、准备提交、需要阶段交付、用户要求验收或审查时必须读取。

## PR TEMPLATE

```markdown
## 本次目标

## 用户可见变化

## 修改范围

## 删除的原入口

## 数据影响

## 规则 / 工作流影响

## 防硬编码说明

## 数据联动说明

## 临时产物清理

## 项目护栏检查

## 测试结果
- 后端：
- 前端：
- 浏览器验收：

## 截图或记录

## 风险

## 回滚方案
```

## NEVER COMPLETE IF

NEVER mark complete if:

- 没有浏览器验收。
- 使用假数据冒充真实功能。
- 只做了后端测试。
- 模板逻辑硬编码。
- 主数据联动未验证。
- 修改了测试来掩盖失败。
- 没有说明用户如何检查。
- 被替换的原入口、原引用、原文件未删除。
- 临时文件、运行产物、依赖目录进入提交。
- 项目护栏检查失败或未运行。
- 实现任务直接在 `main` 开发。
- 失败分支未删除。

## DONE

用户可见功能同时满足：

- 需求有文档来源。
- 前端能操作。
- 后端真实生效。
- 数据能联动。
- 无硬编码模板逻辑。
- 后端自检通过。
- 浏览器验收通过。
- 项目护栏检查通过。
- 原入口和原引用已清理。
- 临时产物已清理。
- 用户有复查路径。
- 风险和回滚明确。
