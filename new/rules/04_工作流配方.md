# 04_工作流配方

## READ WHEN

任务涉及工作流、节点、画布、步骤卡片、Agent 生成流程、试跑、启用、停用、版本回滚时必须读取。

## READ ALSO

实现工作流内部结构、节点、执行、画布或版本时，读取：

`../docs/19_工作流配方详细设计规范.md`

MUST:

- 工作流配方是结构化数据，不是静态流程图。
- Agent 生成草稿，工具层确定性执行。
- 用户确认后才能启用正式版本。
- 已启用工作流再次运行时，不依赖 Agent 再次推理。
- 工作流变更生成后续版本。
- 执行过程在前端可见。

NEVER:

- 把工作流散落在多个按钮事件里。
- 让 Agent 每次临场决定正式报表怎么生成。
- 覆盖原版本。
- 跳过样本试跑。

## MIN FIELDS

```text
name
inputs
nodes
edges
node_params
data_sources
field_mappings
filters
calculations
validations
outputs
permissions
sample_runs
version
confirmed_by
```

## CHECK

- 是否能显示步骤卡片？
- 是否能进入画布查看节点？
- 是否能样本试跑？
- 是否能保存版本？
- 是否能回滚？
- 是否能定位失败节点？
