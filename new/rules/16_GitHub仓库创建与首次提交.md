# 16_GitHub仓库创建与首次提交

## READ WHEN

创建 GitHub 私有仓库、初始化目标开发目录、首次提交、首次 PR 前读取。

## READ ALSO

`../docs/13_GitHub私有仓库创建与首次提交手册.md`

## MUST

- 仓库必须是 Private。
- 用户确认仓库名称和目标开发目录。
- 用户完成 GitHub 授权。
- 首次提交只包含初始化文件和已确认文档。
- `.gitignore` 必须先创建。
- 首次 PR 不实现产品功能。
- 首次 PR 合并后，下一任务先实现项目护栏检查脚本。

## NEVER

- 不要创建 Public 仓库。
- 不要复制当前工作区整包。
- 不要提交运行数据、依赖目录、日志、缓存、临时文件。
- 不要跳过 PR。
- 不要在首次 PR 中实现产品功能。

## DONE

```text
仓库可见性：Private
目标目录：
首次提交内容：
.gitignore：
PR：
用户检查路径：
```
