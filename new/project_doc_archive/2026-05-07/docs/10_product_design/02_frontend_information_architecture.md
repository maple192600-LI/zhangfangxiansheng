# 首页与信息架构定稿说明

> 状态：needs-rewrite。本文仅作页面结构参考，当前权威入口见 `../README.md` 和 `../00_governance/06_document_authority_audit.md`。

## 一、文档目的

本文档用于固定当前已经确认的前端首页定位、左侧信息架构、资金板块层级关系和主要页面职责。

后续前端开发、页面讨论、模块接入，应以本文件为直接依据。

## 二、整体原则

当前前端界面遵循以下原则：

第一，首页独立置顶。

第二，首页服务于出纳工作开场。

第三，资金板块是第一阶段真正要做透的主战场。

第四，首页、工作台、结果表、综合报表之间职责分离。

第五，左侧导航与右侧内容结构保持稳定，不轻易变动。

## 三、左侧主导航正式结构

当前左侧主导航固定为：

- 首页
- 资金板块
- OCR识别
- 贷款管理
- 预算管理
- AI智能体
- 系统设置

第一阶段真正进入主开发闭环的，是“首页”和“资金板块”。

OCR识别、贷款管理、预算管理、AI智能体、系统设置中除必要基础配置外，当前阶段以目录占位和结构预埋为主。

## 四、首页正式定位

### 1. 首页角色

首页是出纳打开系统后的总控工作台。

首页用于展示：

- 今日工作总览
- 待办追踪
- 快捷入口
- 系统提醒

### 2. 首页不承担的职责

首页不承担以下职责：

- 不直接做批量录入
- 不直接做大表明细处理
- 不直接替代日报处理台
- 不做领导汇报分层页面
- 不做复杂 BI 平台

### 3. 首页下的右侧标签页

首页当前固定四个页签：

- 工作总览
- 待办追踪
- 快捷入口
- 系统提醒

### 4. 首页内容重心

首页的内容重心是“今天先做什么、现在做到哪一步、哪里有异常、可以从哪里快速进入处理”。

## 五、资金板块正式结构

资金板块当前固定为三个二级区：

- 工作台
- 资金日报表
- 资金综合报表

### 1. 工作台

工作台负责接数据、处理数据、维护数据。

下含：

- 网银导入
- 手工流水
- 手动维护
- 上传结果预览

#### 网银导入

负责银行流水文件上传、识别、导入、初步处理。

#### 手工流水

负责系统内快速录入，以及 Excel 手工总表上传入口。

#### 手动维护

负责归属修正、字段修正、摘要修正、异常修正和补录修正。

#### 上传结果预览

负责展示待处理项、已处理项、待确认项、重复疑似项，并提供进入基础数据表前的最后预览。

### 2. 资金日报表

资金日报表负责结果展示。

下含：

- 基础数据表
- 现金日记账
- 账户余额表
- 收入明细表
- 支出明细表

#### 基础数据表

是所有后续资金结果表的统一基础底座。

#### 现金日记账

展示现金类资金载体的结果视图。

#### 账户余额表

展示各账户余额结果。

#### 收入明细表、支出明细表

展示收入与支出的明细结果视图。

### 3. 资金综合报表

资金综合报表负责基础汇总视图。

下含：

- 主要账户余额表
- 月末盘点表
- 资金周报
- 资金月报
- 资金年报

第一阶段重点保证基础结构和入口存在，复杂汇总逻辑可分步深化。

## 六、OCR识别的当前定位

OCR识别区当前作为二阶段能力预埋。

下含：

- 发票识别
- 合同识别
- 回单识别
- 付款凭证识别
- 发票台账
- 合同台账

第一阶段只需要预留入口、结构和未来挂载位置，不进入主链路深做。

## 七、贷款管理、预算管理、AI智能体的当前定位

### 1. 贷款管理

作为后续功能预埋。

### 2. 预算管理

当前保留“资金计划”入口，占位即可。

### 3. AI智能体

当前保留目录与结构，用于未来承接规则 agent、日报 agent、费用 agent、收入 agent 等能力。

第一阶段不进入复杂多 Agent 主流程。

## 八、系统设置的当前定位

系统设置当前主要承接以下内容：

- 数据中心
- 规则中心
- 异常中心
- AI配置
- 用户和权限
- 备份恢复
- 操作日志

其中第一阶段真正需要进入开发闭环的重点是：

- 账户数据管理
- 报表模板管理
- 银行流水规则
- AI 配置
- 备份恢复
- 操作日志

## 九、首页与资金板块之间的关系

首页负责总控与提醒。

资金板块负责实际处理与结果生成。

首页上的快捷入口和待办追踪，最终都应进入资金板块中的具体处理页。

后续开发中，不应把大量实际处理功能塞回首页。

## 十、页面职责边界

### 1. 首页

负责总览和进入处理。

### 2. 工作台

负责导入、录入、修正、预览。

### 3. 基础数据表

负责接住所有已标准化基础记录。

### 4. 结果表

负责结果展示、筛选、导出、打印。

### 5. 综合报表

负责基础汇总视图。

### 6. 设置区

负责主数据、规则、模板、备份、AI 配置等管理。

## 十一、第一阶段前端开发重点

第一阶段前端真正需要优先做透的页面包括：

- 首页 / 工作总览
- 首页 / 待办追踪
- 首页 / 快捷入口
- 资金板块 / 工作台 / 网银导入
- 资金板块 / 工作台 / 手工流水
- 资金板块 / 工作台 / 手动维护
- 资金板块 / 工作台 / 上传结果预览
- 资金板块 / 资金日报表 / 基础数据表
- 资金板块 / 资金日报表 / 现金日记账
- 资金板块 / 资金日报表 / 账户余额表
- 系统设置 / 数据中心 / 账户数据管理
- 系统设置 / 规则中心 / 银行流水规则
- 系统设置 / AI配置 / API KEY配置
- 系统设置 / 备份恢复
- 系统设置 / 操作日志

## 十二、视觉与交互提示

首页继续采用多维表格工作台风格。

主操作页继续以表格为主，不做花哨炫技型页面。

手工流水页和基础数据表页应继续围绕 Excel 感、筛选感、核对感来设计。

## 十三、结论

当前前端信息架构已经足够稳定。

后续开发时，首页独立置顶、资金板块三层结构、工作台四步链路、基础数据表为底座，这几件事都应视为正式定稿内容。
